# Спецификация Markdown-уроков AlgoNeoCourse

Этот документ описывает актуальный формат уроков и файлов курса для репозитория `AlgoNeoCource`.

## 1. Файл курса

Каждый курс описывается отдельным JSON-файлом:

- `course1.json`
- `course2.json`
- или любой другой `courseN.json`

Минимальная структура:

```json
{
  "title": "Название курса",
  "description": "Короткое описание",
  "lessons": [
    { "id": "l2m2y1", "title": "Основы синтаксиса C#", "file": "lessons2/m2/y1.md" }
  ]
}
```

Правила:

- `id` урока должен быть уникален в рамках курса;
- `file` хранит путь к Markdown-файлу относительно корня репозитория;
- для новых курсов рекомендуется схема `courseN.json` + `lessonsN/`.

## 2. Один урок = один `.md` файл

Каждый элемент из `lessons` указывает на отдельный `.md` файл.

Пример:

```json
{ "id": "l2m2y3", "title": "Операторы", "file": "lessons2/m2/y3.md" }
```

## 3. Слайды

Слайды разделяются строкой `---`.

```md
# Введение

Краткий контекст урока.

---

## Теория

Основное объяснение.
```

Правила:

- между блоками оставляйте пустые строки;
- используйте `#` для титульного слайда урока;
- используйте `##` для обычных слайдов;
- придерживайтесь принципа "один слайд — одна мысль".

## 4. Поддерживаемые элементы

### 4.1 Обычный Markdown

Рабочий минимум:

- заголовки;
- абзацы;
- списки;
- fenced code blocks;
- изображения и видео через `![](...)`.

Не стоит рассчитывать на корректную работу сложного HTML и тяжёлых таблиц.

### 4.2 Медиа

```md
![](images/example.png)
![](videos/demo.mp4)
![](https://example.com/video.webm)
```

Поддерживаются:

- относительные пути от текущего `.md`;
- проектные пути `Assets/...` и `Packages/...`;
- голое имя файла, если его нужно найти по проекту;
- внешние `http://` и `https://` URL.

Примечания:

- GIF как GIF не воспроизводятся, поэтому лучше сразу использовать `.mp4`;
- при включённой авто-конвертации GIF может быть преобразован в `.mp4` автоматически;
- внешние хостинги иногда отвечают `403` или `429`, в этом случае медиа может быть временно пропущено.

## 5. Квизы

Квизы добавляются через fenced-блок `quiz`.

```quiz
id: l2m2y3-q1
kind: single
text: Какой оператор выполняет проверку условия?
answers:
  - text: if
    correct: true
  - text: for
  - text: while
```

Поддерживаемые типы:

- `single`
- `multiple`
- `truefalse`

Правила:

- каждый квиз должен иметь уникальный `id`;
- один вопрос лучше размещать на одном слайде;
- пояснение к ответу лучше выносить на следующий слайд;
- для `multiple` явно пишите в тексте, что нужно выбрать несколько ответов.

Прогресс квизов сохраняется локально в Unity-плагине автоматически.

## 6. Автопроверки

Автопроверки оформляются через fenced-блок `check`.

````md
```check
rules:
  - object_exists: "Player"
  - component_exists: { object: "Player", type: "Rigidbody" }
  - filename: "PlayerController.cs"
  - contains: "public class PlayerController"
```
````

Поддерживаемые базовые правила:

- `object_exists`
- `component_exists`
- `filename`
- `contains`

Рекомендации:

- проверяйте ключевые условия, а не полностью весь код;
- крупные задания разбивайте на несколько маленьких проверок;
- если проверяете файл, сначала указывайте `filename`, затем `contains`.

## 7. Unity-ссылки

Для открытия ассетов и других действий используйте `unity://` ссылки.

```md
[Открыть префаб](unity://open?path=Assets/Prefabs/Player.prefab)
```

## 8. Горячие клавиши и поведение окна

В окне курса:

- `Left` / `Right` — переход между слайдами;
- `R` — перезагрузка текущего урока;
- `O` — открыть текущий `.md`.

Окно открывается через:

```text
Tools -> AlgoNeoCourse -> Open Course Window
```

Если в `Quiz Settings` включена защита навигации, переход вперёд блокируется, пока незавершённый квиз не будет закрыт.

## 9. Front matter (Удалено)

Раздел удален по требованию автора.

## 10. Минимальный шаблон урока

````md
# Название урока

Короткое вступление.

---

## Теория

Объяснение и пример.

---

## Вопрос

```quiz
id: lesson-q1
kind: single
text: Вопрос по теме?
answers:
  - text: Верный ответ
    correct: true
  - text: Неверный ответ
```

---

## Пояснение

Почему ответ именно такой.

---

## Практика

```check
rules:
  - filename: "Example.cs"
  - contains: "Debug.Log"
```

---

## Итоги

- Что изучили
- Что закрепили
````

## 11. Ограничения

- не полагайтесь на сложный встроенный HTML;
- не делайте гигантские стены текста;
- не используйте неуникальные `id` квизов;
- не вставляйте GIF, если можно сразу положить MP4.

## 12. Чек-лист перед коммитом

- запись в `courseN.json` добавлена;
- путь в `file` существует;
- все разделители `---` стоят корректно;
- у каждого квиза уникальный `id`;
- все медиа доступны;
- практики содержат `check`, если задание должно проверяться автоматически;
- урок открывается и пролистывается в `AlgoNeoCourse`.
