# Справочник по медиа

Актуальные правила по изображениям, видео и GIF для уроков `AlgoNeoCourse`.

## Базовый синтаксис

Любое медиа вставляется через Markdown-нотацию изображения:

```md
![](путь-или-url)
```

Тип определяется по расширению.

## Что поддерживается

### Изображения

Рекомендуемые форматы:

- `.png`
- `.jpg`
- `.jpeg`

Допустимые, но нежелательные:

- `.gif` — только как статичное изображение;
- `.bmp`, `.tiff` — обычно слишком тяжёлые.

### Видео

Лучший вариант:

- `.mp4`

Также поддерживаются:

- `.mov`
- `.webm`
- `.avi`
- `.wmv`
- `.mpeg`
- `.mpg`
- `.m4v`
- `.ogv`
- `.asf`
- `.dv`
- `.vp8`

## Способы указания путей

### Относительный путь

Путь считается относительно папки текущего `.md` файла.

```md
![](images/pic.png)
![](./images/pic.png)
![](../shared/banner.png)
```

Это лучший вариант для локальных медиа урока.

### Проектный путь Unity

```md
![](Assets/UI/Icons/logo.png)
![](Packages/com.company.package/Documentation~/image.png)
```

Используйте, если файл уже живёт внутри Unity-проекта или пакета.

### Голое имя файла

```md
![](logo.png)
```

Плагин попробует найти файл по имени в проекте. Это удобно для прототипирования, но плохо подходит для долгосрочной поддержки, если одинаковые имена могут повторяться.

### Внешний URL

```md
![](https://example.com/image.jpg)
![](https://example.com/demo.mp4)
```

Подходит для больших файлов и централизованного хостинга.

## Практические рекомендации

### Для изображений

- используйте PNG для UI, схем и скриншотов;
- используйте JPEG для фотографий и тяжёлых картинок;
- не кладите текст только в картинку, если его можно оставить обычным текстом;
- придерживайтесь понятных ASCII-имен файлов.

### Для видео

- по возможности всегда используйте MP4;
- держите ролики короткими и точечными;
- если видео длинное, лучше разбить его на несколько фрагментов;
- большие ролики лучше хранить на внешнем хостинге или в релизных ассетах, а не в самом уроке.

## GIF

### Как это работает сейчас

- GIF сам по себе в окне курса не проигрывается как анимация;
- без конвертации GIF будет показан как статичная картинка;
- если включена авто-конвертация, плагин может преобразовать GIF в `.mp4` и обновить слайд.

### Что рекомендуется

Лучшее решение — сразу использовать `.mp4`, а не `.gif`.

Причины:

- быстрее загрузка;
- меньше сюрпризов;
- не нужен этап конвертации;
- стабильнее итоговый результат.

### Если всё же используете GIF

В Unity:

1. Откройте `Tools -> AlgoNeoCourse -> Settings -> Open Course Settings`.
2. Включите авто-конвертацию GIF в MP4.
3. При необходимости укажите путь к `ffmpeg`, если не подходит встроенная конфигурация.

После этого плагин:

- скачает GIF, если он внешний;
- закеширует конвертированный результат;
- обновит отображение слайда после успешной конвертации.

Если операция долгая, у пользователя есть возможность отмены.

## Проблемы с внешними URL

Некоторые CDN и сайты отдают:

- `403 Forbidden`
- `429 Too Many Requests`

В такой ситуации:

- медиа может временно не загрузиться;
- плагин обычно делает паузу перед повторными попытками;
- для стабильных уроков лучше использовать собственный хостинг или локальные файлы.

## Рекомендуемая структура файлов

```text
lessons2/
  m2/
    y1.md
    images/
      y1-step-1.png
      y1-step-2.png
    videos/
      y1-demo.mp4
  shared/
    banner.png
```

## Хорошие и плохие имена файлов

Хорошо:

```text
player-movement-demo.mp4
unity-inspector-object.png
if-statement-flow.png
```

Плохо:

```text
IMG_001.png
новый файл (2).png
video1.mp4
```

## Отладка

### Картинка не отображается

Проверьте:

- путь;
- расширение;
- регистр символов;
- существование файла;
- доступность URL.

### Видео не воспроизводится

Проверьте:

- что это прямой файл, а не страница;
- формат, желательно MP4;
- размер файла;
- что ссылка не требует авторизации.

### GIF остался картинкой

Это ожидаемо, если авто-конвертация выключена или не сработала. Для важных материалов лучше заранее подготовить MP4.
