# Введение в UI и события (EventSystem) — подготовка к Модулю 2, Урок 4

**Цель урока:** понять, как Unity обрабатывает нажатия на кнопки и какие компоненты нужны для работы интерфейса.

---

## Почему кнопка не работает без EventSystem?

Когда вы добавляете кнопку в игру (например, «Кликни меня!»), может показаться: создал кнопку → подключил событие `onClick` → всё заработало! Но если после запуска игры нажатия игнорируются — скорее всего, проблема в **EventSystem**.

---

## Теория 1: Что такое EventSystem?

**EventSystem** (Система событий) — это невидимый компонент Unity, который «слушает» ввод игрока и передаёт его UI-элементам. Без него кнопки, ползунки и другие элементы интерфейса **не реагируют на клики**.

Как это работает:
1. Игрок нажимает кнопку мышью или касается экрана пальцем.
2. `EventSystem` получает этот сигнал от системы ввода (Input).
3. `EventSystem` определяет, какой UI-элемент находится под курсором/пальцем.
4. Событие передаётся этому элементу → кнопка «понимает», что её нажали и вызывает действие из `onClick`.

> **Аналогия:** EventSystem — это диспетчерская служба: вы звоните (клик), диспетчер определяет адрес (UI-элемент) и отправляет бригаду (выполняет действие).

## Пояснение (EventSystem)

- EventSystem — это «посредник» между действиями игрока (клик, касание клавиши) и UI-элементами. Без него кнопки не срабатывают при нажатии мышью или пальцем.
- Рисованием интерфейса занимается Canvas + GraphicRaycaster, а звуком — AudioSource/AudioListener.

---

## Теория 2: Как создать EventSystem?

В Unity есть два способа получить работающую систему событий:

**Способ 1 (автоматический):** Когда вы создаёте UI-элемент через меню **GameObject → UI → Button**, Unity автоматически проверяет наличие EventSystem. Если её нет — создаёт сам! Это самый простой путь для новичков.

**Способ 2 (ручной, если нужно):**
Меню: **GameObject → UI → Event System**. На сцене появится пустой объект `EventSystem` с двумя компонентами:
- `EventSystem` — основной компонент системы событий;
- `Standalone Input Module` — модуль ввода для клавиатуры/мыши.

> **Важно:** на одной сцене должна быть только ОДНА EventSystem! Две и более вызовут конфликты (события будут дублироваться или работать некорректно).

## Пояснение (автоматическое создание)

- При создании любого UI-элемента через меню GameObject → UI, Unity автоматически создаёт недостающие компоненты: если нет Canvas — создаст его; если нет EventSystem — тоже создаст. Поэтому новичкам обычно не нужно создавать их вручную.
- Ручное создание нужно только в особых случаях (например, при использовании нескольких камер или мобильных платформ с сенсорным экраном).

## Теория 3: Что такое onClick и как оно связано с EventSystem?

Когда вы добавляете кнопку на Canvas, у неё есть компонент `Button`. В инспекторе кнопки внизу есть раздел **Click Events** — это список действий, которые выполнятся при клике.

Как подключить действие к кнопке:
1. Перетащите объект со скриптом в слот объекта (Object).
2. Выберите метод из выпадающего списка (`Dynamic` → `No Function` → ваш метод).
3. Теперь при нажатии кнопки вызовётся этот метод — НО только если работает EventSystem!

```csharp
using UnityEngine;

public class ClickerGame : MonoBehaviour
{
    public int score = 0;

    // Этот метод можно подключить к кнопке через onClick:
    void AddScore()
    {
        score++;
        Debug.Log($"Очки: {score}");
    }
}
```

> **Почему это работает:** EventSystem получает клик → находит кнопку под курсором → вызывает событие `onClick` → выполняет метод `AddScore`. Без EventSystem цепочка обрывается на первом шаге.

## Практика: Проверьте, что у вас есть EventSystem

1. Откройте сцену вашего проекта (Clicker Game).
2. Посмотрите в **Hierarchy** — найдите объект с именем `EventSystem`.
3. Если его нет → создайте кнопку через меню GameObject → UI → Button и проверьте снова: Unity должна создать её автоматически.
4. Нажмите Play, кликните по кнопке на экране — убедитесь, что событие срабатывает (появится сообщение в консоли).

```check

  - object_exists: "EventSystem"
  - component_exists:

```
