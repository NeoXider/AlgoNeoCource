# Модуль 5, Урок 3: Ответы на вопросы (Методы и return)

**Цель урока:** научиться писать методы, которые не только выполняют действие, но и **возвращают ответ** (например, `bool`), и применить их в логике боя: пауза между выстрелами, проверка состояния жизни врага и разрушения Стены.

Во время игры скрипты постоянно задают вопросы: можно ли уже стрелять арбалету? Жив ли гоблин? Проиграна ли игра (разрушена ли стена)? 
Писать длинные математические условия `if` внутри логики движения или стрельбы неудобно. Гораздо лучше, когда на эти вопросы отвечают короткие «умные» методы.

---

## Теория 1: Метод с возвращаемым значением

Вспомните наш первый метод `void TakeDamage()`. Слово `void` означает «пустота». Это значит, что метод просто делает работу (вычитает здоровье) и ничего нам в ответ не говорит.

Но метод может не только что-то делать, но и **возвращать** значение как калькулятор! Тогда при его создании вместо `void` мы пишем тип ответа (например, `bool` для ответа да/нет), а внутри метода обязательно даём команду выдать ответ с помощью слова `return`.

Примеры:
- `bool CanShoot()` — можно ли сейчас стрелять?
- `bool IsAlive()` — живо ли существо?
- `int GetMaxHealth()` — выдай мне цифру максимального здоровья.

А потом мы просто пишем в условии: `if (CanShoot()) { ... }`. И код читается легко, как обычный английский текст!

```csharp
    void Update()

                targetPoint.position,
                moveSpeed * Time.deltaTime);

    }

    // --- НОВЫЙ КОД ---
    // Метод, который отвечает на вопрос
    public bool IsAlive()
    {
        return currentHealth > 0;
    }
```

## Теория 2: Слово `return` как прерыватель

Вы могли заметить странность. В методе `IsAlive()` мы использовали `return true`, то есть вернули значение ответа.
Но в самом начале `Update()` мы написали `if (!IsAlive()) return;` — то есть просто `return;` в пустоту. Почему там нет значения?

Всё очень просто: сам движок Unity создал метод `Update()` изначально с типом **`void`**. Он не имеет права отдавать значения. 
В методах типа `void` команда `return;` означает "экстренный выход"!
Она работает так же, как кнопка "Выйти из комнаты". Как только скрипт натыкается на `return;`, он мгновенно отменяет выполнение всего, что написано ниже (гоблин перестаёт шагать!), и завершает работу скрипта на текущий кадр.

## Практика 2: Переписываем таймер Арбалета

Вспомните 1-й урок. Там мы написали громоздкое условие выстрела в Арбалете: `if (Input.GetMouseButton(0) && Time.time >= nextShootTime)`.
Давайте применим ООП и сделаем код профессиональным!

**Задача:**
1. Откройте скрипт `CrossbowShooter.cs`.
2. Создайте метод `bool CanShoot()`, который будет проверять, остыл ли арбалет (текущее время больше или равно `nextShootTime`).
3. В методе `Update` используйте архитектуру "Ранний выход": если левая кнопка НЕ нажата или если стрелять нельзя `!CanShoot()` — сразу сделайте экстренный `return;`.
4. Если код дошел до последней строчки `Update` и не прервался на `return` — значит можно свободно вызывать `Shoot()` и обновлять `nextShootTime`. 

**Пример кода:**

```csharp
    void Update()
    {
        Vector3 mousePos = Camera.main.ScreenToWorldPoint(Input.mousePosition);
        Vector2 dir = mousePos - transform.position;
        float angle = Mathf.Atan2(dir.y, dir.x) * Mathf.Rad2Deg;
        transform.rotation = Quaternion.Euler(0, 0, angle);

        // --- ИЗМЕНЕННЫЙ КОД (Ранний выход) ---
        // Если кнопка мыши НЕ нажата - отбой, нечего тут делать
        if (!Input.GetMouseButton(0)) return;
        
        // Если пушка не остыла - отбой!
        if (!CanShoot()) return;

        // Если скрипт дожил до этой строчки — 100% можно стрелять!
        nextShootTime = Time.time + shootCooldown;
        Shoot();
    }

    // --- НОВЫЙ КОД ---
    // Метод проверки остывания пушки
    public bool CanShoot()
    {
        return Time.time >= nextShootTime;
    }
```

## Теория 3: Проигрыш и Атака Стены

Дойдя до левой границы экрана, гоблины упираются в нашу Стену. 
Чтобы сделать игру полноценной, у Стены должна быть своя точка защиты и запас прочности. И если он опустится до нуля, игра должна спросить `HasLost()?` и сообщить нам о поражении!

Кроме того, враги не должны разбиваться об стену! Они должны остановиться, и, как и Арбалет, использовать таймер выстрела, чтобы наносить стене урон, пока их самих не убьют.

## Пояснение

### Часть 2: Урон по базе

**Задача:**
1. Создайте скрипт `DefensePoint.cs` и повесьте его на спрайт вашей Стены (`Target`). Убедитесь что на Стене есть `BoxCollider2D` (без галочки Trigger!).
2. Напишите там здоровье (`health = 10`), метод `TakeDamage(int damage)` и логический метод проверки поражения `HasLost()`.
3. Откройте скрипт врага `EnemyUnit.cs` и добавьте поля для таймера атаки `attackCooldown = 1f` и цель `targetWall`.
4. В `EnemyUnit.cs` добавьте метод `OnCollisionEnter2D`. Если гоблин стукнулся о Стену — он должен сохранить её в `targetWall`.
5. И обновите `Update` гоблина: если `targetWall` найден — стоим на месте и атакуем её по таймеру! Если нет — идём влево.

**Пример кода скрипта Стены (DefensePoint.cs):**
```csharp
using UnityEngine;

public class DefensePoint : MonoBehaviour
{
    public int health = 10;

    public void TakeDamage(int p_damage)
    {
        health -= p_damage;
        
        if (HasLost())
        {
            Debug.Log("СТЕНА ПАЛА! ИГРА ПРОИГРАНА!");
            // Позже мы добавим вызов меню поражения
        }
    }

    public bool HasLost()
    {
        return health <= 0;
    }
}
```

**Обновляем EnemyUnit.cs (Остановка и атака):**
```csharp
    // --- НОВЫЕ ПОЛЯ В НАЧАЛО EnemyUnit.cs ---
    private DefensePoint targetWall;
    public float attackCooldown = 1f;
    private float nextAttackTime;

    // --- ОБНОВЛЕННЫЙ Update ---
    void Update()
    {
        if (!IsAlive()) return; 

        // Если мы дошли до стены — бьём её по таймеру!
        if (targetWall != null)
        {
            if (Time.time >= nextAttackTime)
            {
                nextAttackTime = Time.time + attackCooldown;
                targetWall.TakeDamage(1);
            }
        }
        else // Если нет стен — просто идём влево
        {
            transform.Translate(Vector3.left * moveSpeed * Time.deltaTime);
        }
    }

    // --- НОВЫЙ КОД ФИЗИКИ ---
    void OnCollisionEnter2D(Collision2D coll)
    {
        // Пытаемся найти компонент стены на том, с кем жестко столкнулись
        DefensePoint basePoint = coll.gameObject.GetComponent<DefensePoint>();
        
        if (basePoint != null)
        {
            targetWall = basePoint; // Запоминаем стену (перестаём двигаться и начинаем атаковать!)
        }
    }
```
