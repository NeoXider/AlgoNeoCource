# Модуль 5, Урок 3: Методы с ответом — `return`, база и проигрыш

**Цель урока:** научиться писать методы, которые возвращают ответ, и применить их в игре: можно ли стрелять, жив ли враг, разрушена ли база.

К концу урока у базы будет здоровье, а враг сможет нанести ей урон, если дойдёт до неё.

---

## Зачем метод должен отвечать?

Иногда метод не просто делает действие.
Он отвечает на вопрос:

- можно ли стрелять?
- жив ли враг?
- разрушена ли база?

Для ответа "да/нет" используют `bool`.

```csharp
public bool IsAlive()
{
    return currentHealth > 0;
}
```

## Пояснение (`bool` как ответ)

Метод с типом `bool` похож на вопрос.
Он отвечает только двумя вариантами: `true` или `false`.

Поэтому имена вроде `IsAlive()`, `CanShoot()` и `HasLost()` читаются почти как обычные вопросы.

---

## Теория 1: `return` возвращает ответ

Слово `return` означает: "верни результат и закончи метод".

Например:

```csharp
return currentHealth > 0;
```

Здесь C# сначала проверяет условие.
Если здоровье больше нуля, вернётся `true`.
Если нет — `false`.

## Пояснение (`!IsAlive()`)

`IsAlive()` отвечает: "враг жив?"

Знак `!` переворачивает ответ:

- `IsAlive()` — жив;
- `!IsAlive()` — не жив.

Так код становится короче и понятнее.

## Пояснение (условие смерти)

`if (!IsAlive())` читается так: "если враг уже не жив".

Внутри этого `if` можно выполнять действие смерти: удалить врага, начислить награду или запустить эффект.
В этом уроке оставляем самый простой вариант — удалить врага.

## Теория 2: `return;` может остановить метод

В методах типа `void` нельзя вернуть значение.
Но можно написать просто:

```csharp
return;
```

Это значит: "закончить метод прямо сейчас".

Так удобно делать ранний выход: если условие не подходит, не выполнять остальной код.

## Пояснение (допуск к выстрелу)

`return;` внутри `void`-метода работает как стоп-кнопка.

Если стрелять нельзя, метод заканчивается сразу.
Остальной код ниже не выполняется, и нам не нужно писать большой вложенный `if`.

## Теория 3: База тоже объект со своей ролью

База должна хранить своё здоровье.
Она сама лучше всех знает, разрушена она или нет.

Для этого создадим `DefensePoint`.

Его роль:

- хранить здоровье базы;
- получать урон;
- отвечать на вопрос `HasLost()`.

## Пояснение (роль базы)

Здоровье базы — это состояние самой базы.
Поэтому его хранит `DefensePoint`.

Враг не решает, проиграл ли игрок.
Он только наносит урон, а база сама отвечает на вопрос `HasLost()`.

## Практика 3: Скрипт `DefensePoint`

### Шаг 1: Создайте скрипт базы

Создайте скрипт `DefensePoint.cs` и повесьте его на объект `DefensePoint`.

### Шаг 2: Добавьте здоровье и проверку проигрыша

```csharp
using UnityEngine;

public class DefensePoint : MonoBehaviour
{
    public int health = 10;

    public void TakeDamage(int damage)
    {
        health -= damage;

        if (HasLost())
        {
            Debug.Log("База разрушена!");
        }
    }

    public bool HasLost()
    {
        return health <= 0;
    }
}
```

## Пояснение (`HasLost()`)

Метод:

```csharp
public bool HasLost()
{
    return health <= 0;
}
```

можно читать как вопрос: "мы уже проиграли?"

Если здоровье базы `0` или меньше, ответ `true`.
Если здоровье ещё есть, ответ `false`.

## Пояснение (почему враг исчезает)

Мы делаем простую версию игры.
Если враг дошёл до базы:

1. база получает урон;
2. враг исчезает;
3. игрок понимает, что пропустил угрозу.

Это проще, чем заставлять врага стоять у базы и атаковать по таймеру.
Такую механику можно добавить позже, когда основа уже понятна.

---

## Если база не получает урон

Проверьте сцену:

- `DefensePoint` стоит на пути врага;
- у `DefensePoint` есть `Collider2D`;
- у `Enemy` есть `Collider2D` и `Rigidbody2D`;
- хотя бы один коллайдер работает как trigger;
- метод `OnTriggerEnter2D` находится именно в `EnemyUnit`.

Если объекты визуально пересекаются, но сообщения нет, значит Unity не видит trigger-событие.

---

## Проверьте сцену

1. У `DefensePoint` должен быть `Collider2D` с `Is Trigger`.
2. У `Enemy` должен быть `Collider2D` с `Is Trigger`.
3. У `Enemy` должен быть `Rigidbody2D` с `Gravity Scale = 0`.
4. Враг должен идти влево.
5. Если враг дошёл до базы, в Console должно появиться сообщение `База разрушена!`, когда здоровье упадёт до нуля.
