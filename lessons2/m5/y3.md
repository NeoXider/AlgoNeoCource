# Модуль 5, Урок 3: Методы и return

**Цель урока:** научиться писать методы, которые не только выполняют действие, но и **возвращают ответ** (например, `bool`), и применять их в логике боя: пауза между выстрелами, проверка жизни врага и проигрыш.

Игра постоянно задаёт вопросы: можно ли стрелять, жив ли враг, проиграл ли игрок. Удобно, когда на это отвечают короткие методы.

---

Постоянно проверять всё длинными условиями в одном месте неудобно.
Лучше завести методы, которые возвращают понятный результат: да или нет.

---

## Теория 1: Метод с возвращаемым значением

Метод может не только что-то делать, но и **возвращать** значение. Тогда в объявлении вместо `void` пишут тип результата, например `bool`, а в конце метода — `return ответ`.

Примеры:

- `bool CanShoot()` — можно ли сейчас стрелять (например, прошла ли пауза).
- `bool IsAlive()` — жив ли объект (здоровье > 0).
- `bool HasLost()` — проиграна ли игра (HP базы <= 0).

Такие методы удобно использовать в условиях: `if (CanShoot()) Shoot();`

### Часть 2: Пауза между выстрелами (CanShoot)

**Задача:**

1. В `CrossbowShooter.cs` добавьте поля:
   - `public float attackCooldown = 0.7f;` — пауза между выстрелами в секундах.
   - `float currentCooldown;` — сколько осталось ждать (в начале можно 0).
2. Добавьте метод `bool CanShoot()`, который возвращает `true`, если `currentCooldown <= 0f`, иначе `false`.
3. В `Update()` каждый кадр уменьшайте `currentCooldown` на `Time.deltaTime`.
4. Вместо прямого вызова `Shoot()` при нажатии клавиши сделайте метод `TryShoot()`: если `!CanShoot()`, вызовите `return;` и выйдите. Иначе вызовите `Shoot()` и присвойте `currentCooldown = attackCooldown;`
5. В `Update()` при нажатии клавиши вызывайте `TryShoot()`.

**Пример:**

```csharp
public float attackCooldown = 0.7f;
float currentCooldown;

void Update()
{
    currentCooldown -= Time.deltaTime;
    if (Input.GetKeyDown(shootKey))
        TryShoot();
}

public void TryShoot()
{
    if (!CanShoot()) return;
    Shoot();
    currentCooldown = attackCooldown;
}

public bool CanShoot()
{
    return currentCooldown <= 0f;
}
```

## Теория 2: Зачем нужен return

- `return` отправляет результат из метода наружу.
- Вызвавший код может использовать этот результат в `if`: `if (CanShoot()) ...`
- Ранний выход: в методе с `void` можно писать `return;` без значения — выполнение метода закончится, остальной код не выполнится. Так удобно выходить из метода, если условие не выполнено: `if (!CanShoot()) return;`

## Практика 2: Проверка жизни врага (IsAlive)

**Задача:**

1. В `EnemyUnit.cs` добавьте метод `public bool IsAlive()`, который возвращает `currentHealth > 0`.
2. В `TakeDamage` после уменьшения здоровья проверяйте: `if (!IsAlive()) Destroy(gameObject);` вместо проверки `currentHealth <= 0` напрямую.

Так логика «враг мёртв» будет в одном месте и станет читаемее.

```csharp
public bool IsAlive()
{
    return currentHealth > 0;
}

public void TakeDamage(int damage)
{
    currentHealth -= damage;
    if (!IsAlive())
        Destroy(gameObject);
}
```

---

Теперь игра не просто выполняет команды, а принимает решения: можно ли стрелять, жив ли враг, проиграл ли игрок. Всё это спрятано в коротких методах с понятными именами.

---

## Почему короткие методы удобнее длинных условий?

Если писать одно длинное условие в середине кода, его сложнее читать и проверять.
Когда логика вынесена в метод с названием `CanShoot()` или `IsAlive()`, код читается как фраза: «если можно стрелять — стреляй».

---

## Теория 3: Короткие методы делают код чище

- Короткий метод проще проверить и объяснить.
- Его легче переиспользовать.
- Название метода заменяет комментарий: `if (!CanShoot()) return;` сразу понятно без чтения тела `CanShoot()`.

Особенно это полезно в боевой логике и при проверке состояния игры.
