# Модуль 5, Урок 2: Классы и объекты — здоровье и урон

**Цель урока:** разложить логику по разным скриптам (арбалет, стрела, враг, точка защиты) и добавить первые характеристики: здоровье и урон.

Сейчас мы переходим от «сцена просто работает» к «каждый объект хранит свои данные и делает свою часть работы».

---

Если здоровье, урон, движение и выстрел держать в одном скрипте, код быстро запутается.
Удобнее, когда у врага — своё здоровье, у стрелы — свой урон, у точки защиты — своё HP.

---

## Теория 1: Класс и объект на примере игры

**Класс** — это описание «каким должен быть» объект (какие у него поля и методы).
**Объект** — это конкретный экземпляр на сцене, созданный по этому классу.

Один и тот же класс можно использовать для разных объектов:

- Класс «враг» описывает движение и здоровье.
- Конкретный враг на сцене — это объект.
- У разных врагов могут быть разные значения одних и тех же полей (например, разное здоровье).

## Практика 1: Скрипт врага (EnemyUnit)

**Задача:**

1. Создайте скрипт `EnemyUnit.cs`. В нём объявите поля:
   - `public int maxHealth = 3;`
   - `public int currentHealth = 3;`
   - `public float moveSpeed = 2f;`
   - `public Transform targetPoint;` — точка, к которой враг идёт (позже это будет точка защиты).
2. В `Start()` присвойте `currentHealth = maxHealth;`
3. В `Update()` двигайте врага к `targetPoint` через `Vector3.MoveTowards`, если `targetPoint` не null.
4. Добавьте метод `public void TakeDamage(int damage)` — уменьшайте `currentHealth` на `damage`. Если `currentHealth <= 0`, вызовите `Destroy(gameObject);`

**Пример кода:**

```csharp
using UnityEngine;

public class EnemyUnit : MonoBehaviour
{
    public int maxHealth = 3;
    public int currentHealth = 3;
    public float moveSpeed = 2f;
    public Transform targetPoint;

    void Start()
    {
        currentHealth = maxHealth;
    }

    void Update()
    {
        if (targetPoint != null)
        {
            transform.position = Vector3.MoveTowards(
                transform.position,
                targetPoint.position,
                moveSpeed * Time.deltaTime);
        }
    }

    public void TakeDamage(int damage)
    {
        currentHealth -= damage;
        if (currentHealth <= 0)
            Destroy(gameObject);
    }
}
```

Создайте на сцене объект врага, повесьте на него `EnemyUnit`, задайте `targetPoint` (например, пустой объект у точки защиты). Сохраните врага как префаб.

## Теория 2: Поля и данные объекта

**Поля** — это переменные, которые хранят данные объекта. У каждого объекта на сцене — свои копии этих полей.

- У врага: `currentHealth`, `maxHealth`, `moveSpeed`.
- У точки защиты: своё здоровье.
- У стрелы: скорость полёта и урон.

Ссылки на другие объекты (например, `targetPoint`) тоже хранятся в полях и настраиваются в инспекторе.

### Часть 2: Точка защиты (DefensePoint)

**Задача:**

1. Создайте скрипт `DefensePoint.cs` с полями:
   - `public int maxHealth = 10;`
   - `public int currentHealth = 10;`
2. В `Start()` присвойте `currentHealth = maxHealth;`
3. Добавьте метод `public void TakeDamage(int damage)` — уменьшайте `currentHealth` на `damage`. (Проверку проигрыша добавим позже.)

Повесьте скрипт на объект, который будет «базой» (точкой защиты). Этот же объект можно использовать как `targetPoint` для врагов.

```csharp
using UnityEngine;

public class DefensePoint : MonoBehaviour
{
    public int maxHealth = 10;
    public int currentHealth = 10;

    void Start()
    {
        currentHealth = maxHealth;
    }

    public void TakeDamage(int damage)
    {
        currentHealth -= damage;
    }
}
```

## Практика 3: Урон от стрелы

**Задача:**

1. В `BoltProjectile.cs` добавьте поле `public int damage = 1;`
2. В `CrossbowShooter.cs` добавьте поле `public int boltDamage = 1;` В методе `Shoot()` после `Instantiate` получите компонент созданной стрелы и присвойте ему урон: `newBolt.damage = boltDamage;` (тип переменной при Instantiate — `BoltProjectile`).
3. В `BoltProjectile.cs` добавьте коллайдер с включённым **Is Trigger** на префабе стрелы. На префабе врага тоже должен быть коллайдер (Is Trigger по необходимости).
4. В `BoltProjectile` в методе `OnTriggerEnter(Collider other)` найдите у `other` компонент `EnemyUnit`. Если он есть, вызовите `TakeDamage(damage)` и удалите стрелу: `Destroy(gameObject);`

**Пример фрагмента BoltProjectile:**

```csharp
public int damage = 1;

void OnTriggerEnter(Collider other)
{
    EnemyUnit enemy = other.GetComponent<EnemyUnit>();
    if (enemy != null)
    {
        enemy.TakeDamage(damage);
        Destroy(gameObject);
    }
}
```

Враг при столкновении с базой может наносить урон точке защиты: в `EnemyUnit` в `OnTriggerEnter` проверьте `DefensePoint` у `other` и вызовите у неё `TakeDamage(damageToBase)`, затем уничтожьте врага.

> **Напоминание:** метод `OnTriggerEnter(Collider other)` вызывается Unity, когда объект с коллайдером-триггером входит в контакт с другим коллайдером. Через `other` мы получаем тот объект, с которым столкнулись. `other.GetComponent<EnemyUnit>()` возвращает скрипт врага на этом объекте (или null, если его нет).

## Теория 3: Разделение ответственности

Скрипт арбалета не должен хранить здоровье врага. Скрипт врага не должен управлять кнопками улучшений.

Когда у каждого объекта своя роль, игру проще расширять:

- `EnemyUnit` — здоровье и движение к цели.
- `DefensePoint` — здоровье базы.
- `BoltProjectile` — полёт и нанесение урона при столкновении с врагом.
