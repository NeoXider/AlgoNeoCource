# Модуль 5, Урок 2: Классы, объекты и движение врагов

**Цель урока:** разложить логику по разным скриптам (добавить первого врага) и познакомить с первыми характеристиками: здоровьем, скоростью и коллизиями.

Сейчас мы переходим от «сцена просто работает» к концепции ООП (Объектно-Ориентированное Программирование), где каждый объект хранит свои данные и делает свою отдельную часть работы.

---

Если здоровье, урон и движение врагов держать в одном скрипте с Арбалетом, код моментально превратится в неподдерживаемую кашу.
Удобнее, когда у врага — своё здоровье, у болванки — свой урон, у стены — свой запас прочности.

## Теория 1: Класс и объект на примере игры

**Класс** — это чертёж или описание «каким должен быть» объект (какие у него характеристики и поведение).
**Объект** — это конкретный экземпляр на сцене, созданный по этому чертежу.

Один и тот же чертёж (скрипт) можно использовать для разных объектов на сцене:

- Класс «враг» описывает, что любой враг умеет ходить и имеет здоровье.
- Конкретный гоблин на сцене — это объект.
- У разных гоблинов (объектов) могут быть разные значения одного и того же здоровья.

Характеристики (здоровье, скорость) в коде называются **полями**. Поле — это переменная внутри класса, которая хранит данные конкретного объекта.

## Теория 2: Прямолинейное движение

Гоблины должны наступать на Стену!
Поскольку наша Стена занимает большую часть левой стороны экрана, гоблинам не нужно искать конкретный пиксель, им достаточно просто идти по прямой непрерывно влево.

Для этого мы используем уже знакомую команду `transform.Translate`, передавая туда математический вектор идеального движения влево (`Vector3.left`), помноженный на вашу скорость.

## Теория 3: Создание метода ранения

Чтобы враг мог терять здоровье, ему нужен свой собственный **метод** (действие). Назовем его `TakeDamage(int damage)`.
Этот метод должен принимать число (урон от конкретной стрелы), вычитать это число из текущего здоровья, а затем проверять: не упало ли здоровье до нуля? Если да — уничтожаем гоблина!

---

## Практика 3: Физическое уничтожение

**Задача:**

1. Напишите метод ранения `TakeDamage` в скрипте врага.
2. Сохраните объект `Enemy` как **префаб** (выделите его и перетащите в папку окна Project). 

**Пример кода:**

```csharp
    // --- НОВЫЙ КОД ---
    // Метод (публичный, чтобы стрела могла к нему обратиться)
    public void TakeDamage(int damage)
    {
        currentHealth -= damage;
        
        // Если здоровье кончилось — убиваем объект
        if (currentHealth <= 0)
        {
            Destroy(gameObject);
        }
    }
```

## Теория 4: Столкновение двух миров (`OnTriggerEnter2D`)

У нас есть стрела (Триггер), и у нас есть Гоблин (Триггер).
Когда они накладываются друг на друга на экране, Unity сам вызывает магический физический метод `OnTriggerEnter2D(Collider2D collision)`.

Вся сила ООП раскрывается здесь! Оказавшись внутри этого столкновения, стрела может проверить: "А есть ли у того, с кем я столкнулась, скрипт `EnemyUnit`?". Если есть — стрела вызывает его функцию `TakeDamage`, наносит урон, а сама уничтожается (чтобы не пролетать гоблина насквозь).

### Часть 2: Реализация столкновения

**Задача:**
1. Откройте скрипт **стрелы** (`BoltProjectile.cs`).
2. Добавьте переменную урона `public int damage = 1;`.
3. Добавьте физический метод столкновения `OnTriggerEnter2D`.
4. Внутри найдите врага, нанесите ему урон и уничтожьте стрелу.

**Пример кода:**

```csharp
using UnityEngine;

public class BoltProjectile : MonoBehaviour
{
    public float moveSpeed = 36f;
    public float lifeTime = 6f;
    
    // --- НОВОЕ ПОЛЕ ---
    public int damage = 1;

    void Start()
    {
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        transform.Translate(Vector3.right * moveSpeed * Time.deltaTime);
    }

    // --- НОВЫЙ КОД ---
    void OnTriggerEnter2D(Collider2D coll)
    {
        // Пытаемся взять компонент EnemyUnit с того объекта, в который мы влетели
        EnemyUnit enemy = coll.GetComponent<EnemyUnit>();
        
        // Если враг действительно существует
        if (enemy != null)
        {
            enemy.TakeDamage(damage); // Наносим урон
            Destroy(gameObject);      // Уничтожаем саму стрелу
        }
    }
}
```

