## Теория 1: Класс и объект

**Класс** — это описание.
Например, `EnemyUnit.cs` описывает, что умеет враг.

**Объект** — это конкретный враг на сцене.
По одному классу можно создать много врагов:

- у каждого будет своё здоровье;
- каждый будет двигаться сам;
- каждый сможет умереть отдельно.

Класс похож на чертёж врага.

Объект — это уже конкретный враг на сцене.
Из одного класса можно получить много врагов, и у каждого будет своё здоровье.

## Теория 2: Поля — память объекта

Враг должен помнить свои параметры:

- сколько у него максимального здоровья;
- сколько здоровья осталось сейчас;
- с какой скоростью он идёт.

Для этого в классе нужны поля.

`maxHealth` можно менять в Inspector.
`currentHealth` лучше хранить внутри скрипта, потому что это текущее состояние конкретного врага.

## Почему `currentHealth` не public

`maxHealth` — настройка врага. Её удобно менять в Inspector.

`currentHealth` — внутреннее состояние во время игры.
Если открыть его всем скриптам, любой объект сможет случайно поменять здоровье врага.

Поэтому здесь логика такая:

- снаружи вызывают `TakeDamage(...)`;
- сам враг решает, сколько здоровья осталось;
- сам враг решает, когда ему исчезнуть.

Это уже маленькая **инкапсуляция**: важное состояние не меняют напрямую, а меняют через понятный метод.

В учебном коде мы пока оставляем часть настроек `public`, чтобы быстро видеть их в Inspector.
В более аккуратном Unity-коде часто используют `[SerializeField] private`: поле видно в Inspector, но другие скрипты не трогают его напрямую.

## Практика 2: Скрипт `EnemyUnit`

### Шаг 1: Добавьте поля врага

Добавьте код:

```csharp
using UnityEngine;

public class EnemyUnit : MonoBehaviour
{
    public int maxHealth = 3;
    public float moveSpeed = 2f;

    private int currentHealth;

    void Start()
    {
        currentHealth = maxHealth;
    }

    void Update()
    {
        transform.Translate(Vector3.left * moveSpeed * Time.deltaTime);
    }

    public void TakeDamage(int damage)
    {
        currentHealth -= damage;

        if (currentHealth <= 0)
        {
            Destroy(gameObject);
        }
    }
}
```

```check
rules:
  - filename: "EnemyUnit.cs"
  - contains: "public int maxHealth"
  - contains: "private int currentHealth"
  - contains: "public void TakeDamage"
  - contains: "Destroy(gameObject)"
```

---

## Как работает враг

> **По шагам:**
>
> 1. В `Start()` враг получает полное здоровье.
> 2. В `Update()` он каждый кадр идёт влево.
> 3. `TakeDamage(int damage)` уменьшает здоровье.
> 4. Если здоровье стало `0` или меньше, враг удаляется.

Это важная идея ООП: враг сам отвечает за своё здоровье.

## Теория 3: Как стрела узнаёт, что попала?

У стрелы есть коллайдер.
У врага есть коллайдер.

Когда они пересекаются, Unity может вызвать специальный метод:

```csharp
OnTriggerEnter2D(Collider2D other)
```

Внутри этого метода стрела проверит:

- есть ли у объекта, в который она попала, скрипт `EnemyUnit`;
- если есть — вызвать `TakeDamage(...)`;
- после попадания удалить саму стрелу.

```quiz
  - text: Только при нажатии мыши
```

## Пояснение (trigger-событие)

`OnTriggerEnter2D(...)` вызывает Unity, когда два 2D-коллайдера пересеклись и настроены как trigger.

Это удобно: стреле не нужно каждый кадр искать врага.
Она просто реагирует на событие столкновения.

## Практика 3: Добавьте урон в `BoltProjectile`

### Шаг 1: Откройте скрипт стрелы

Откройте `BoltProjectile.cs` и добавьте поле `damage` и метод `OnTriggerEnter2D`.

### Шаг 2: Добавьте проверку попадания

```csharp
using UnityEngine;

public class BoltProjectile : MonoBehaviour
{
    public float speed = 7f;
    public float lifeTime = 4f;
    public int damage = 1;

    void Start()
    {
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        transform.Translate(Vector3.right * speed * Time.deltaTime);
    }

    void OnTriggerEnter2D(Collider2D other)
    {
        EnemyUnit enemy = other.GetComponent<EnemyUnit>();

        if (enemy != null)
        {
            enemy.TakeDamage(damage);
            Destroy(gameObject);
        }
    }
}
```

```check
  - contains: "GetComponent<EnemyUnit>()"
  - contains: "enemy.TakeDamage(damage)"
```

---

## Пояснение (`enemy != null`)

Строка:

```csharp
EnemyUnit enemy = other.GetComponent<EnemyUnit>();
```

пытается найти `EnemyUnit` на объекте, в который попала стрела.

Если компонент найден, `enemy != null`.
Значит, перед стрелой действительно враг, и ему можно нанести урон.

Если компонент не найден, стрела попала не во врага.

## Пояснение (`null` после GetComponent)

`GetComponent<EnemyUnit>()` пытается найти компонент врага на объекте столкновения.

- если компонент есть, вернётся ссылка на `EnemyUnit`;
- если компонента нет, вернётся `null`.

Проверка `enemy == null` защищает код от попытки нанести урон объекту, который не является врагом.

## Если `OnTriggerEnter2D` не вызывается

Это не всегда ошибка в коде.
Чаще всего не хватает одной настройки в сцене:

- нет `Collider2D` у стрелы или врага;
- не включён `Is Trigger`;
- нет `Rigidbody2D`;
- объект пули на сцене отличается от префаба, который создаёт арбалет.

Проверяйте именно префаб `Bolt`, потому что в игре создаётся он.
