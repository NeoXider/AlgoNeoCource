# Модуль 5, Урок 4: Глобальный контроллер (GameManager и Экономика)

**Цель урока:** ввести общий управляющий объект всей игры (`GameManager`), добавить экономику (золото за победу над врагами) и создать первые UI-кнопки для прокачки арбалета.

После этого урока проект перестанет быть просто "стрелялкой" и начнёт ощущаться как полноценная игра с прогрессом: награда за врагами и трата валюты на усиления характеристик.

---

До сих пор каждый скрипт занимался только своим делом: Арбалет стрелял, Враги наступали. 
Но кто отвечает за общий счёт, количество заработанного золота и появление меню поражения?
Такие глобальные вещи удобно собирать в одном невидимом объекте — управляющем **GameManager**.

---

## Теория 1: Зачем игре нужен GameManager

Часть данных относится не к одному конкретному объекту, а ко всей сессии игры целиком:

- текущее богатство (золото или монеты);
- состояние игры (идёт бой или выбило окно "Вы проиграли!");
- ссылки на элементы интерфейса (панель прокачки, текст монет).

Их удобно хранить в одном управляющем объекте — Директоре игры. Тогда любой гоблин, умерев, сможет сказать Директору: «Эй, я умер, начисли игроку 5 золотых!», а Директор сам разберётся, что к чему.

## Пояснение

- GameManager — это «информационный центр» для общих данных и логики. Если положить деньги игрока внутрь гоблина, то после смерти гоблина удалятся и все накопленные деньги! Поэтому нужен отдельный безопасный скрипт.

### Часть 2: Кодим хранилище

**Задача:**

1. Создайте пустой объект на сцене (клик правой кнопкой в иерархии -> `Create Empty`). Назовите его `GameManager`.

**Пример кода:**

```csharp
using UnityEngine;

public class GameManager : MonoBehaviour
{
    // Сколько у нас сейчас золота

    public void AddCurrency(int value)

        currentCurrency += value;
        Debug.Log("Денег стало: " + currentCurrency);
    }
}
```

## Теория 2: Связь через Инспектор и Награда

GameManager готов к начислению денег. Но кто должен вызывать метод `AddCurrency`?
Конечно же гоблин, в момент своей смерти от стрелы!

Чтобы гоблин мог связаться с GameManager, гоблину нужна **ссылка** на него. Мы создадим пустое поле типа `GameManager` внутри `EnemyUnit`, а в Юнити просто перетащим объект менеджера в это поле!

---

## Практика 2: Вытрясаем золото из врагов

**Задача:**

1. Откройте скрипт `EnemyUnit.cs`.
2. Добавьте поле для ссылки `public GameManager gameManager;` и поле для награды `public int reward = 5;`.
3. В методе `TakeDamage`, прямо перед разрушением `Destroy(gameObject)`, обратитесь к менеджеру и добавьте деньги.
4. **Важно:** Сохраните скрипт, выделите префаб вашего врага в папке (или на сцене) и перетащите объект `GameManager` из Иерархии прямо в новое поле `Game Manager` внутри скрипта врага! Убедитесь что ссылка установилась.

**Пример измененного кода `EnemyUnit.cs`:**
```csharp
    // --- НОВЫЕ ПОЛЯ ---
    public GameManager gameManager;
    public int reward = 5;

    public void TakeDamage(int damage)
    {
        currentHealth -= damage;
        
        if (currentHealth <= 0)
        {
            // Перед смертью начисляем награду (если менеджер подключен)
            if (gameManager != null)
            {
                gameManager.AddCurrency(reward);
            }
            Destroy(gameObject);
        }
    }
```

```check
rules:
  - filename: "EnemyUnit.cs"
  - contains: "public GameManager gameManager"
  - contains: "gameManager.AddCurrency(reward)"
```

Запустите игру и убейте гоблина. В консоли внизу экрана вы должны увидеть белую надпись: `Денег стало: 5`!

## Теория 3: Текстовый интерфейс (UI)

Для отображения интерфейса в Юнити используется система `Canvas` — это невидимый прозрачный слой, который накладывается поверх камеры. 
Чтобы управлять текстом из кода, нам потребуется специальная библиотека **`using TMPro;`** и тип данных `TMP_Text`.

---

## Практика 3: Рисуем золото

**Задача:**
1. Нажмите правой кнопкой в иерархии -> `UI` -> `Text - TextMeshPro`. Юнити может попросить импортировать TMP Essentials — соглашайтесь нажимая Import Essentials.
2. Юнити сам создаст огромный `Canvas` и внутри него текстовый объект (назовите его `CurrencyText`). Поместите текст в левый верхний угол через Инспектор.
3. Откройте скрипт `GameManager.cs`. В самом верху файла добавьте `using TMPro;`.
4. Внутри класса добавьте ссылку `public TMP_Text currencyText;`.
5. Напишите метод `UpdateUI()` и вызывайте его каждый раз, когда прибавляются деньги!
6. Вернитесь в Юнити, выделите объект `GameManager` и перетащите ваш текст со сцены в поле `Currency Text`.

**Пример `GameManager.cs`:**
```csharp
using UnityEngine;
using TMPro; // --- ОБЯЗАТЕЛЬНО ДЛЯ UI ---

public class GameManager : MonoBehaviour
{
    public int currentCurrency = 0;
    
    // --- НОВОЕ ПОЛЕ ---
    public TMP_Text currencyText;

    void Start()
    {
        UpdateUI(); // Обновляем баланс при старте игры
    }

    public void AddCurrency(int value)
    {
        currentCurrency += value;
        UpdateUI(); // --- ОБНОВЛЯЕМ ТЕКСТ ПОСЛЕ ПРИБЫЛИ ---
    }

    // --- НОВЫЙ МЕТОД ---
    public void UpdateUI()
    {
        if (currencyText != null)
        {
            currencyText.text = "Золото: " + currentCurrency;
        }
    }
}
```

```check
rules:
  - object_exists: "CurrencyText"
    has_component: "TextMeshProUGUI"
  - filename: "GameManager.cs"
  - contains: "using TMPro"
  - contains: "currencyText.text ="
```

Убейте гоблина и полюбуйтесь: цифра на экране меняется сама! Экономика заработала.

## Теория 4: Кнопки Улучшений

Зачем нам деньги, если их нельзя тратить? Давайте дадим игроку возможность покупать улучшения (например, скорострельность арбалета или урон).
Это делается через интерфейсные **Кнопки (`Button`)**.

У каждой кнопки есть особый блок-событие `On Click ()`. В него можно перетащить любой объект со сцены (например, наш GameManager), выбрать публичный метод внутри него — и этот метод выполнится ровно тогда, когда игрок физически кликнет по кнопке. Это работает как настоящая магия!

### Часть 2: Магазин улучшений

**Задача:**
1. В скрипт `GameManager.cs` добавьте ссылку на наш Арбалет (`public CrossbowShooter crossbowShooter;`) и цену прокачки (`public int upgradeCost = 15;`).
2. Напишите публичный метод `BuyAttackSpeed()`. Он должен проверить: если денег хватает — списать оплату, улучшить `shootCooldown` у арбалета (например, вычесть 0.1) и обновить UI.
3. В Юнити создайте Кнопку (`правой кнопкой -> UI -> Button - TextMeshPro`), расположите её внизу экрана. Разверните кнопку, выделите дочерний Text и напишите там: "Скорость (15g)".
4. Выделите саму кнопку. Прокрутите свойства вниз до `On Click ()`. Нажмите плюс `+`.
5. Перетащите в левую нижнюю ячейку объект `GameManager`.
6. Раскройте выпадающий список справа (No Function): выберите `GameManager` -> `BuyAttackSpeed()`. 
7. Не забудьте перетащить объект Арбалета со сцены в поле `Crossbow Shooter` внутри вашего Менеджера!

**Дополнение в GameManager.cs:**
```csharp
    // --- НОВЫЕ ПОЛЯ ---
    public CrossbowShooter crossbowShooter;
    public int upgradeCost = 15;

    // --- НОВЫЙ МЕТОД ПОКУПКИ ---
    public void BuyAttackSpeed()
    {
        if (currentCurrency >= upgradeCost)
        {
            currentCurrency -= upgradeCost;        // Списываем деньги
            crossbowShooter.shootCooldown -= 0.1f; // Улучшаем арбалет (уменьшаем задержку)
            UpdateUI();                            // Обновляем текст золота
            Debug.Log("Скорострельность куплена!");
        }
    }
```
