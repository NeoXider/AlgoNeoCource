**Цель урока:** ввести общий управляющий объект игры (GameManager), добавить валюту за победу над врагами и первые улучшения через UI (урон и скорость атаки).

После этого урока проект начнёт ощущаться как игра с прогрессом: награда за врагов и трата валюты на усиления.

Арбалет отвечает за стрельбу, враг — за движение и здоровье.
Но кто отвечает за общий счёт, валюту и кнопки улучшений?
Такие вещи удобно собрать в одном объекте — **GameManager**.

## Теория 1: Зачем игре нужен GameManager

Часть данных относится не к одному объекту, а ко всей игре:

- текущая валюта (монеты);
- состояние «игра идёт» или «игра окончена»;
- ссылки на UI (тексты, кнопки).

Их удобно хранить в одном управляющем объекте. Тогда арбалет, враги и UI обращаются к нему, когда нужно добавить валюту, обновить текст или проверить проигрыш.

## Практика 1: Создание GameManager

**Задача:**

1. Создайте пустой объект на сцене и назовите его `GameManager`.
2. Создайте скрипт `GameManager.cs` и повесьте на этот объект.
3. Добавьте поля:
   - `public CrossbowShooter crossbowShooter;`
   - `public DefensePoint defensePoint;`
   - `public TMP_Text currencyText;` (для вывода валюты; нужен `using TMPro;`)
   - `public int currentCurrency;` (текущие монеты)
4. В методе `UpdateUI()` обновляйте текст валюты: `currencyText.text = "Монеты: " + currentCurrency;` (или аналогично). Вызовите `UpdateUI()` в `Start()` и при изменении валюты.
5. Добавьте метод `public void AddCurrency(int value)` — увеличивайте `currentCurrency` на `value` и вызывайте `UpdateUI()`.

Создайте на сцене Canvas, добавьте текст (TextMeshPro) для валюты и перетащите его в поле `currencyText` в GameManager.

**Пример:**

```csharp
using UnityEngine;
using TMPro;

public class GameManager : MonoBehaviour
{
    public CrossbowShooter crossbowShooter;
    public DefensePoint defensePoint;
    public TMP_Text currencyText;

    public int currentCurrency;

    void Start()
    {
        UpdateUI();
    }

    public void AddCurrency(int value)
    {
        currentCurrency += value;
        UpdateUI();
    }

    void UpdateUI()
    {
        if (currencyText != null)
            currencyText.text = "Монеты: " + currentCurrency;
    }
}
```

## Теория 2: Валюта за победу над врагом

Когда враг уничтожен, игрок должен получить награду. Логично вызывать это из скрипта врага: перед `Destroy(gameObject)` сообщить GameManager о награде.

Для этого у врага нужны:

- поле `public int reward = 2;` — сколько валюты дать за убийство;
- ссылка на GameManager (поле `public GameManager gameManager;`, назначить в инспекторе).

В `TakeDamage`, когда враг погибает (`!IsAlive()`), вызовите `gameManager.AddCurrency(reward);` (если `gameManager != null`), затем `Destroy(gameObject);`

## Как сделать улучшения понятными?

Кнопка улучшения должна работать просто: нажал → потратил валюту → получил усиление. Проверку «хватает ли монет» удобно делать в GameManager и вызывать из кнопки через событие `onClick`.

---

## Теория 3: UI и улучшения

- Кнопка в Unity при нажатии вызывает метод (например, `GameManager.TryBuyDamageUpgrade`). Метод настраивается в инспекторе в компоненте Button → On Click.
- Игра проверяет: если валюты достаточно — списывает её и усиливает параметр; если нет — ничего не делает (или выводит подсказку).
- Улучшения лучше делать по одному эффекту: одна кнопка — урон, другая — скорость атаки.

### Часть 2: Улучшение урона и скорости атаки

**Задача:**

1. В `CrossbowShooter.cs` добавьте два метода:
   - `public void UpgradeDamage(int value)` — увеличивает `boltDamage` на `value`;
   - `public void UpgradeAttackSpeed(float value)` — уменьшает `attackCooldown` на `value` (но не ниже минимального, например 0.15f, чтобы игра не сломалась).
2. В `GameManager.cs` добавьте:
   - `public int damageUpgradeCost = 5;` и `public int attackSpeedUpgradeCost = 5;`
   - `public bool CanSpendCurrency(int price)` — возвращает `currentCurrency >= price;`
   - `public void TryBuyDamageUpgrade()` — если `CanSpendCurrency(damageUpgradeCost)` — вычесть стоимость из `currentCurrency`, вызвать `crossbowShooter.UpgradeDamage(1);`, обновить UI.
   - `public void TryBuyAttackSpeedUpgrade()` — аналогично для скорости атаки.
3. Создайте две UI-кнопки. В On Click первой назначьте `GameManager.TryBuyDamageUpgrade`, второй — `GameManager.TryBuyAttackSpeedUpgrade`.
4. Проверьте: при достаточной валюте кнопки работают, урон и скорость атаки меняются.

**Пример методов в GameManager:**

```csharp
public int damageUpgradeCost = 5;
public int attackSpeedUpgradeCost = 5;

public bool CanSpendCurrency(int price)
{
    return currentCurrency >= price;
}

public void TryBuyDamageUpgrade()
{
    if (!CanSpendCurrency(damageUpgradeCost)) return;
    currentCurrency -= damageUpgradeCost;
    crossbowShooter.UpgradeDamage(1);
    UpdateUI();
}

public void TryBuyAttackSpeedUpgrade()
{
    if (!CanSpendCurrency(attackSpeedUpgradeCost)) return;
    currentCurrency -= attackSpeedUpgradeCost;
    crossbowShooter.UpgradeAttackSpeed(0.1f);
    UpdateUI();
}
```
