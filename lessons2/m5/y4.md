# Модуль 5, Урок 4: `GameManager` — деньги, UI и одно улучшение

**Цель урока:** добавить общий управляющий объект, который хранит деньги игрока, обновляет UI и покупает одно понятное улучшение.

К концу урока враги будут давать монеты, а игрок сможет купить усиление урона стрел.

---

## Зачем нужен `GameManager`?

Не все данные принадлежат одному врагу или одной стреле.

Например:

- сколько монет у игрока;
- сколько стоит улучшение;
- что показывать в UI.

Эти данные удобно хранить в отдельном объекте `GameManager`.

## Пояснение (роль GameManager)

`GameManager` хранит то, что относится ко всей партии, а не к одному объекту.

Монеты игрока — общие.
Они не принадлежат конкретной стреле или конкретному врагу.

Поэтому `GameManager` становится местом, где хранится общее состояние игры.

Console видит разработчик.

Для UI-текста используем `TextMeshPro` и тип `TMP_Text`.

`GameManager` будет обновлять текст каждый раз, когда меняется количество монет.

## Пояснение (текст как ссылка)

`TMP_Text` — это ссылка на надпись в UI.

Так игрок видит результат, а не только сообщение в Console.

## Практика 3: Текст монет

### Шаг 1: Создайте UI-текст

1. Создайте UI-текст `CurrencyText`.
2. Откройте `GameManager.cs`.
3. Добавьте `using TMPro;`.
4. Добавьте поле `public TMP_Text currencyText;`.
5. Добавьте метод `UpdateCurrencyText()`.

### Шаг 2: Обновите код `GameManager`

Итоговый код:

```csharp
using TMPro;
using UnityEngine;

public class GameManager : MonoBehaviour
{
    public int currency;
    public TMP_Text currencyText;

    void Start()
    {
        UpdateCurrencyText();
    }

    public void AddCurrency(int amount)
    {
        currency += amount;
        UpdateCurrencyText();
    }

    private void UpdateCurrencyText()
    {
        currencyText.text = "Монеты: " + currency;
    }
}
```

## Если монеты есть в Console, но не видно на экране

Проверьте:

- в сцене есть объект `CurrencyText`;
- на нём компонент TextMeshPro;
- `CurrencyText` перетащен в поле `Currency Text` у `GameManager`;
- в Console нет `NullReferenceException`.

Если ошибка падает на `currencyText.text`, значит ссылка `currencyText` пустая.

```csharp
    GameObject boltObject = Instantiate(boltPrefab, shootPoint.position, Quaternion.identity);

    if (bolt != null)
    {
        bolt.damage = boltDamage;

}
```

## Пояснение (почему урон задаёт арбалет)

Стрела живёт недолго: появилась, полетела, попала или исчезла.

А улучшение покупает игрок для арбалета.
Поэтому удобно хранить текущий урон в `CrossbowShooter`, а при создании новой стрелы передавать ей значение:

```csharp
bolt.damage = boltDamage;
```

Так все новые стрелы получают улучшенный урон.

## Практика 5: Покупка улучшения

### Шаг 1: Добавьте поля покупки

Откройте `GameManager.cs`.
Добавьте поля:

```csharp
public CrossbowShooter crossbow;
public int damageUpgradeCost = 10;
```

### Шаг 2: Добавьте метод покупки

Добавьте метод:

```csharp
public void BuyDamageUpgrade()
{
    if (currency < damageUpgradeCost)
    {
        return;
    }

    currency -= damageUpgradeCost;
    crossbow.boltDamage += 1;
    UpdateCurrencyText();
}
```

Если монет не хватает, `return;` тихо завершает покупку.
Это нормальное поведение для простой версии: игрок нажал кнопку, но ресурсов пока недостаточно.
