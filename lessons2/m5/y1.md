# Модуль 5, Урок 1: ООП в Tower Defence — арбалет и первая стрела

**Цель урока:** начать простую Tower Defence-игру и увидеть главную идею ООП: каждый объект отвечает за свою понятную задачу.

К концу урока на сцене будет арбалет, точка выстрела и стрела, которая появляется, летит вправо и сама удаляется.

---

## Что мы строим в этом модуле?

Игрок защищает базу слева.
Арбалет стреляет вправо.
Враги позже будут идти справа налево.

Главная идея модуля:

- арбалет стреляет;
- стрела летит;
- враг получает урон;
- база хранит здоровье;
- `GameManager` хранит общее состояние игры.

Так устроено ООП: у каждого объекта есть своя роль.

---

## ООП без сложных слов

ООП — это не просто “много классов”.
Это способ разложить игру на объекты, у которых есть понятная работа.

В профессиональном программировании у ООП есть четыре важных принципа:

- **абстракция** — прячем лишние детали за понятным действием;
- **инкапсуляция** — не даём менять важные данные как попало;
- **наследование** — создаём похожие классы на основе общего родителя;
- **полиморфизм** — разные объекты могут по-своему выполнять одно действие.

В этом модуле мы начнём с того, что сразу видно в игре:

- объект имеет **роль**;
- объект хранит своё **состояние**;
- объект даёт другим скриптам понятные **методы**.

А профессиональные слова будем привязывать к конкретному коду, когда они реально понадобятся.

## Теория 1: Арбалет не должен знать всё

Арбалет не должен управлять врагами, базой и деньгами.
Его задача проще:

- проверить, нажата ли кнопка мыши;
- создать стрелу в `ShootPoint`;
- подождать небольшую паузу перед следующим выстрелом.

Это и есть хорошая роль объекта: короткая и понятная.

## Пояснение (роль арбалета)

`CrossbowShooter` не должен быть главным начальником всей игры.
У него короткая работа:

- заметить ввод игрока;
- создать стрелу;
- поставить паузу до следующего выстрела.

Когда роль объекта такая понятная, код легче читать и чинить.

## Практика 2: Скрипт `CrossbowShooter`

### Шаг 1: Создайте скрипт арбалета

Создайте скрипт `CrossbowShooter.cs` и повесьте его на объект `Crossbow`.

### Шаг 2: Добавьте код выстрела

Добавьте код:

```csharp
using UnityEngine;

public class CrossbowShooter : MonoBehaviour
{
    public GameObject boltPrefab;
    public Transform shootPoint;
    public float shootCooldown = 0.5f;

    private float nextShootTime;

    void Update()
    {
        if (Input.GetMouseButton(0) && Time.time >= nextShootTime)
        {
            Shoot();
            nextShootTime = Time.time + shootCooldown;
        }
    }

    public void Shoot()
    {
        Instantiate(boltPrefab, shootPoint.position, Quaternion.identity);
    }
}
```

  - contains: "Instantiate(boltPrefab"
```

## Как работает выстрел

> **По шагам:**
>
> 1. `Input.GetMouseButton(0)` проверяет, зажата ли левая кнопка мыши.
> 2. `Time.time >= nextShootTime` проверяет, прошла ли пауза между выстрелами.
> 3. `Shoot()` создаёт стрелу.
> 4. `nextShootTime = Time.time + shootCooldown` назначает время следующего разрешённого выстрела.

Арбалет пока не поворачивается.
Это нормально: сначала делаем понятную механику, потом можно усложнять.

## Пояснение (выстрел и ссылки)

Перед тем как делать красивую стрелу, проверьте простую вещь:

- при клике вызывается `Shoot()`;
- в Inspector у `Crossbow` есть ссылки на `Bolt Prefab` и `Shoot Point`;
- `ShootPoint` стоит справа от арбалета.

Если ссылка пустая, Unity не знает, что именно создавать.

## Пояснение (пауза между выстрелами)

`Update()` вызывается каждый кадр.
Если игрок зажмёт кнопку мыши без паузы, арбалет будет создавать слишком много стрел.

`nextShootTime` хранит момент, когда следующий выстрел снова разрешён.
Это простой способ управлять темпом игры.

## Теория 2: Стрела — отдельный объект

Стрела не должна быть частью арбалета.

У неё своя роль:

- лететь вперёд;
- исчезнуть через несколько секунд;
- позже наносить урон врагу.

Поэтому для стрелы будет отдельный скрипт `BoltProjectile`.

  - text: Потому что отдельный скрипт автоматически делает объект быстрее
```

---

## Пояснение (роль стрелы)

После выстрела стрела уже не должна каждый кадр спрашивать арбалет, что делать.
Она становится отдельным объектом:

- летит вперёд;
- может столкнуться с врагом;
- сама исчезает через время.

Так в ООП объект получает своё поведение, а не просто картинку на сцене.

## Практика 3: Скрипт `BoltProjectile`

### Шаг 1: Создайте объект стрелы

Создайте объект `Bolt`.
Добавьте на него спрайт стрелы и скрипт `BoltProjectile.cs`.

### Шаг 2: Добавьте код движения

```csharp
using UnityEngine;

public class BoltProjectile : MonoBehaviour
{
    public float speed = 7f;
    public float lifeTime = 4f;

    void Start()
    {
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        transform.Translate(Vector3.right * speed * Time.deltaTime);
    }
}
```

## Почему стрела сама удаляется?

Если стрелы не удалять, они будут улетать далеко за экран и оставаться в сцене.
Игрок их уже не видит, но Unity всё ещё хранит эти объекты.

Поэтому у стрелы есть срок жизни:

```csharp
Destroy(gameObject, lifeTime);
```

Это значит: удалить этот объект через `lifeTime` секунд.

---

## Пояснение (удаление объекта)

`Destroy(gameObject, lifeTime)` не удаляет префаб из папки Project.
Он удаляет только конкретную стрелу, которая уже появилась на сцене.

Это нужно, чтобы старые стрелы не копились за экраном и не мешали игре.

## Практика 4: Сделайте префаб стрелы

### Шаг 1: Сделайте префаб

Перетащите объект `Bolt` из Hierarchy в Project, чтобы сделать префаб.

### Шаг 2: Подключите стрелу к арбалету

Выделите `Crossbow`.
В поле `Bolt Prefab` перетащите префаб `Bolt`.
В поле `Shoot Point` перетащите объект `ShootPoint`.

Нажмите Play и зажмите левую кнопку мыши.

## Что должно получиться

При нажатии мыши арбалет создаёт стрелы.
Стрелы летят вправо и исчезают через несколько секунд.

Это уже первая рабочая связка объектов:

- `CrossbowShooter` создаёт;
- `BoltProjectile` двигается и удаляется;
- каждый скрипт делает свою работу.

---

## Если стрела не появляется

Проверьте по порядку:

- `Bolt` сделан префабом;
- префаб перетащен в поле `Bolt Prefab`;
- `ShootPoint` перетащен в поле `Shoot Point`;
- на `Crossbow` висит `CrossbowShooter`;
- в Console нет красных ошибок.

Чаще всего проблема не в `Instantiate`, а в пустой ссылке в Inspector.
