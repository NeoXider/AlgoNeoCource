# Модуль 5, Урок 1: Введение в ООП — Tower Defence

**Цель урока:** подготовить сцену Tower Defence, собрать арбалет и цель и сделать первый рабочий выстрел стрелой.

Это старт нового модуля: часть времени уйдёт на настройку сцены. К концу урока у вас будет первый заметный результат — выстрел.

---

В новой игре будут арбалет, стрела, враги и точка, которую нужно защищать.
Если смешать всё в одном скрипте, код быстро станет нечитаемым.
Поэтому с самого начала будем раскладывать игру по разным объектам и скриптам.

---

## Теория 1: Объект в игре и его задача

В Unity каждый объект на сцене — это **GameObject**. Поведение ему задают **компоненты** и **скрипты**.

Удобно думать так: **один объект = одна понятная задача.**

- Арбалет отвечает за **выстрел**.
- Стрела отвечает за **полёт** и исчезновение.
- Позже враг будет отвечать за движение, а точка защиты — за здоровье базы.

Так код остаётся понятным, а игру — легко дорабатывать.

## Теория 2: Первый выстрел — что нужно в коде

Чтобы стрела «появилась» и полетела, понадобится:

- **Transform** — чтобы знать, где стоит арбалет и куда смотрит точка выстрела (`shootPoint.position`, `shootPoint.rotation`).
- **Instantiate(...)** — создаёт копию префаба (например, стрелы) в заданной позиции и повороте.
- **Destroy(gameObject, lifeTime)** — удаляет объект через заданное время (чтобы стрелы не копились на сцене).

Сначала сделаем маленький рабочий выстрел, потом будем усложнять.

### Часть 2: Скрипт арбалета (CrossbowShooter)

**Задача:**

1. Создайте C# скрипт `CrossbowShooter.cs` и повесьте его на объект `Crossbow`.
2. Добавьте поля:
   - `public Transform shootPoint;` — ссылка на точку вылета (перетащите `ShootPoint` в инспекторе).
   - `public BoltProjectile boltPrefab;` — префаб стрелы (его создадим в следующем шаге).
   - `public KeyCode shootKey = KeyCode.Space;` — клавиша выстрела.
3. В `Update()` при нажатии клавиши вызывайте метод `Shoot()`.
4. В методе `Shoot()` создавайте стрелу: `Instantiate(boltPrefab, shootPoint.position, shootPoint.rotation);`

**Пример кода:**

```csharp
using UnityEngine;

public class CrossbowShooter : MonoBehaviour
{
    public Transform shootPoint;
    public BoltProjectile boltPrefab;
    public KeyCode shootKey = KeyCode.Space;

    void Update()
    {
        if (Input.GetKeyDown(shootKey))
            Shoot();
    }

    public void Shoot()
    {
        Instantiate(boltPrefab, shootPoint.position, shootPoint.rotation);
    }
}
```

## Практика 3: Стрела (BoltProjectile)

**Задача:**

1. Создайте объект стрелы (например, Capsule или удлинённый куб). Добавьте Collider при необходимости. Назовите объект `Bolt`.
2. Создайте скрипт `BoltProjectile.cs` и повесьте на `Bolt`.
3. В скрипте задайте:
   - `public float moveSpeed = 12f;`
   - `public float lifeTime = 3f;`
4. В `Start()` вызовите `Destroy(gameObject, lifeTime);` — стрела удалится через несколько секунд.
5. В `Update()` двигайте стрелу вперёд: `transform.Translate(Vector3.forward * moveSpeed * Time.deltaTime);`
6. Сохраните объект `Bolt` как **префаб** (перетащите из Hierarchy в папку Project). Удалите экземпляр со сцены.
7. На объекте `Crossbow` в компоненте `CrossbowShooter` в поле **Bolt Prefab** перетащите префаб стрелы.

**Пример кода:**

```csharp
using UnityEngine;

public class BoltProjectile : MonoBehaviour
{
    public float moveSpeed = 12f;
    public float lifeTime = 3f;

    void Start()
    {
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        transform.Translate(Vector3.forward * moveSpeed * Time.deltaTime);
    }
}
```

## Как сделать так, чтобы объект делал только своё дело?

Стреле не нужно «знать» всю игру. Ей достаточно: полететь и исчезнуть.
Арбалету не нужно самому лететь — он только создаёт стрелу.
Такой подход называется **разделением ответственности**.

---

## Теория 3: Одна ответственность у объекта

Каждый объект лучше делать **одну** понятную вещь:

- Арбалет создаёт стрелу по нажатию.
- Стрела летит и удаляется через время.
- Позже враг будет двигаться и иметь здоровье, а точка защиты — хранить HP базы.

Если один скрипт будет делать и выстрел, и полёт, и здоровье врага, в нём будет трудно разобраться.
