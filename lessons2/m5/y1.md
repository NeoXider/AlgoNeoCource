**Цель урока:** подготовить 2D-сцену (вид сверху), собрать арбалет и стену, а также шаг за шагом реализовать поворот арбалета за курсором мыши и стрельбу летящими стрелами.

Это старт нового модуля: на этом уроке мы настроим основу для нашей игры. К концу занятия у вас будет готовое открытое поле, стоящий слева арбалет и управление: оружие будет крутиться за мышью и непрерывно стрелять.

В новой игре будут арбалет, летящая стрела, враги и стена, которую нужно защищать. Мы делаем игру с видом сверху (Top-Down).

## Теория 1: Объект в игре и его задача

В Unity каждый объект на сцене — это **GameObject**. Поведение ему задают **компоненты** (SpriteRenderer, Collider) и **скрипты**.

Удобно думать так: **один объект = одна понятная задача.**

- Арбалет отвечает за **прицеливание и выстрел**.
- Стрела отвечает за **полёт** и исчезновение, чтобы не засорять память.
- Позже враг будет отвечать за движение, а стена — за здоровье защиты.

## Теория 2: Где находится мышка?

Чтобы прицелиться, нам нужно знать, куда смотрит игрок. 
Но есть проблема: мышка бегает по **экрану** вашего монитора (он измеряется в пикселях, например, от 0 до 1920), а наша игра существует в **игровом мире** (он измеряется в условных метрах).

Если мы прямо скажем "повернись к позиции мыши", игра нас не поймёт, ведь 1000 пикселей для неё — это где-то на Луне.
Для "перевода" пикселей в игровые метры есть математический спаситель: `Camera.main.ScreenToWorldPoint()`. Эта функция берёт координаты курсора на экране и "проецирует" их на игровую землю.

---

> **Как найти вектор направления (по шагам):**
> 
> ```csharp
> Vector3 mousePos = Camera.main.ScreenToWorldPoint(Input.mousePosition);
> Vector2 dir = mousePos - transform.position;
> ```
> 
> 1. Получаем позицию мыши в игровом мире (переводим пиксели в метры).
> 2. Считаем направление (вектор) к мыши. Правило простое: **вектор = куда - откуда** (то есть `мышь - мы`).

## Теория 3: Как повернуться в 2D (Угол и Atan2)

Мы знаем направление к мыши (вектор `dir`). Теперь нужно заставить арбалет физически повернуться туда!

В 3D играх мы крутили объекты влево-вправо по оси **Y** (подобно балерине). 
Но в 2D (вид сверху) "чистый" поворот — это вращение героя "на столе", то есть по оси **Z**! Нам нужно вычислить угол и применить его.

Для математического расчёта угла из `x` и `y` координат вектора используется мощная формула **арктангенса**, которая в Unity записывается как `Mathf.Atan2(y, x)`. Она высчитывает точный угол поворота в радианах.

---

> **Как работает поворот (по шагам):**
> 
> ```csharp
> float angle = Mathf.Atan2(dir.y, dir.x) * Mathf.Rad2Deg;
> transform.rotation = Quaternion.Euler(0, 0, angle);
> ```
> 
> 1. Скармливаем `y` и `x` направления в `Atan2`.
> 2. Она выдает результат в радианах, поэтому домножаем на `Mathf.Rad2Deg`, переводя угол в удобные нам градусы.
> 3. Заставляем арбалет повернуться ровно на этот `angle` по оси Z!

## Теория 4: Создание снарядов (Instantiate)

Прицел готов! Осталось научить героя стрелять. Мы будем создавать физическую копию стрелы в игре каждый раз, когда игрок нажимает левую кнопку мыши. 

Для непрерывной проверки зажатой кнопки используется команда `Input.GetMouseButton(0)` (где 0 — это левая кнопка). А для клонирования объекта в Unity применяют мощную команду `Instantiate(...)`. Она создаёт копию нужного префаба ровно в тех координатах, которые мы ей передадим.

## Практика 4: Выстрел по клику

**Задача:**

2. В методе `Update` добавьте простую проверку: если зажата левая кнопка — создаём стрелу функцией `Instantiate`.
3. В инспекторе (в Unity) убедитесь, что перетащили дочерний объект `ShootPoint` в одноимённое поле скрипта!

**Пример кода:**

```csharp

    public Transform shootPoint;
    public GameObject boltPrefab;
```

## Теория 5: Таймер перезарядки (Cooldown)

Если вы запустите игру после прошлого шага и зададите пустую болванку в префаб, то увидите проблему: пока вы держите кнопку, создаются тысячи копий каждую мини-секунду!
Чтобы арбалет стрелял с нормальной скоростью, нам нужен **таймер перезарядки** (Cooldown).

Суть таймера:
1. Запоминаем время, когда можно будет выстрелить в следующий раз (`nextShootTime`).
2. Если текущее время игры (`Time.time`) превысило этот таймер — значит, пушка остыла и можно стрелять!
3. Во время выстрела сдвигаем `nextShootTime` в будущее, прибавляя время нашей задержки (`shootCooldown`).

---

## Практика 5: Внедряем таймер

**Задача:**
1. Добавьте переменные таймера: `shootCooldown` и `nextShootTime`.
2. Дополните ваше условие `if`: теперь стрелять можно только если зажата кнопка **И** текущее время `Time.time` больше таймера.
3. Внутри условия обновите таймер на будущее. (Также мы вынесли выстрел в отдельный метод `Shoot()` для красоты и читаемости кода).

**Пример кода:**

```csharp
using UnityEngine;

public class CrossbowShooter : MonoBehaviour
{
    public Transform shootPoint;
    public GameObject boltPrefab;
    
    // --- НОВЫЕ ПЕРЕМЕННЫЕ ---
    public float shootCooldown = 0.5f;
    private float nextShootTime = 0f;

    void Update()
    {
        Vector3 mousePos = Camera.main.ScreenToWorldPoint(Input.mousePosition);
        Vector2 dir = mousePos - transform.position;
        float angle = Mathf.Atan2(dir.y, dir.x) * Mathf.Rad2Deg;
        transform.rotation = Quaternion.Euler(0, 0, angle);

        // --- ИЗМЕНЕННЫЙ КОД ---
        // Стреляем, ЕСЛИ зажата кнопка И пришло время
        if (Input.GetMouseButton(0) && Time.time >= nextShootTime)
        {
            nextShootTime = Time.time + shootCooldown;
            Shoot();
        }
    }

    public void Shoot()
    {
        Instantiate(boltPrefab, shootPoint.position, shootPoint.rotation);
    }
}
```

```check
  - contains: "Time.time >="
  - contains: "nextShootTime ="
```

## Теория 6: Как заставить стрелу лететь?

Стреле достаточно просто постоянно двигаться вперёд. 
Из-за математической особенности поворота (`Atan2`), который мы запрограммировали в начале, "нос" нашего арбалета (и спавнящейся из него стрелы) всегда направлен в сторону локальной оси X. 
Поэтому, чтобы стрела в любой ситуации летела вперёд туда, куда была выпущена, нам достаточно двигать её по вектору **Vector3.right** (что для команды `Translate` означает: "двигайся вдоль своей локальной оси X").

И как вы помните из прошлых модулей, чтобы стрела летела плавно независимо от мощности компьютера, любое физическое передвижение в `Update` нужно обязательно умножать на множитель времени `Time.deltaTime`.

## Теория 7: Спасение памяти компьютера (Garbage Collection)

Сейчас наши стрелы, если их создать, просто летят в бесконечность. Если игрок зажмёт кнопку выстрела на 5 минут, за пределы экрана улетят тысячи болванок. Это очень быстро "забьёт" оперативную память, и игра начнёт зависать.

Главное правило хорошего разработчика: **Всё, что вылетело за экран и больше не влияет на игру, должно быть безжалостно удалено.** 
Мы знаем, что наша стрела пролетает через всё поле примерно за 6 секунд. Значит, мы можем повесить ей "срок годности" прямо в момент рождения. Для этого у функции `Destroy` есть удобный второй параметр — встроенный таймер удаления: `Destroy(gameObject, 6f)`. Удали сам себя через 6 секунд!

## Практика 7: Таймер уничтожения и Префаб

**Задача:**

1. Откройте скрипт `BoltProjectile.cs`.
2. Добавьте переменную времени жизни `lifeTime = 6f`. Внутри старта (`Start()`) вызовите таймер удаления `Destroy`.
3. Сохраните объект `Bolt` как **префаб** (выделите его на сцене и перетащите в папку окна Project внизу). 
4. После того как префаб зафиксировался (объект в иерархии стал синим), **удалите** сам объект `Bolt` со сцены. Он нам больше там не потребуется.
5. Выделите вашего героя (`Crossbow`). В пустом поле **Bolt Prefab** (в компоненте скрипта `CrossbowShooter`) перетащите туда ваш только что созданный префаб!

**Пример кода:**

```csharp
using UnityEngine;

public class BoltProjectile : MonoBehaviour
{
    public float moveSpeed = 36f;
    public float lifeTime = 6f; // НОВОЕ: время жизни

    {
        // НОВОЕ: Снаряд сам себя удалит через lifeTime секунд
        Destroy(gameObject, lifeTime);
    }

    {
        transform.Translate(Vector3.right * moveSpeed * Time.deltaTime);
    }
}
```
