# Модуль 5, Урок 5: Спавнер и волны врагов

**Цель урока:** сделать объект, который создаёт врагов по времени, а затем добавить простую волну сложности.

К концу урока враги будут появляться сами, а игра перестанет зависеть от врагов, расставленных вручную.

---

## Почему нужен спавнер?

Пока врага приходится ставить на сцену руками.
Это неудобно: игра не умеет продолжаться сама.

Спавнер — это отдельный объект с простой ролью:

- ждать нужное время;
- создать врага;
- передать врагу ссылку на `GameManager`;
- повторить.

## Пояснение (роль спавнера)

Спавнер — это не враг и не арбалет.
Его задача отдельная: создавать новых врагов по правилам.

Так мы не смешиваем движение врага, стрельбу и появление врагов в одном скрипте.

---

## Теория 1: Создание по таймеру

Мы уже делали задержку у арбалета.
Спавнер работает похожим образом:

```csharp
if (Time.time >= nextSpawnTime)
{
    SpawnEnemy();
    nextSpawnTime = Time.time + spawnInterval;
}
```

Разница только в том, что арбалет создаёт стрелу, а спавнер создаёт врага.

## Пояснение (таймер спавна)

`Update()` вызывается каждый кадр.
Без таймера спавнер мог бы создавать врагов слишком быстро.

`nextSpawnTime` работает как расписание: раньше этого времени нового врага создавать нельзя.

## Теория 2: Случайная высота

Чтобы враги не появлялись всегда в одной точке, используем `Random.Range`.

```csharp
float y = Random.Range(bottomPoint.position.y, topPoint.position.y);
```

Это выбирает случайное число между нижней и верхней точкой.

Так спавнер остаётся простым, но игра уже выглядит живее.

## Пояснение (`Random.Range`)

`Random.Range(bottom, top)` выбирает случайное число между двумя границами.

В нашем случае это высота появления врага.
Поэтому враги идут с разных линий, а не выстраиваются в одну скучную колонну.

## Практика 2: Скрипт `EnemySpawner`

### Шаг 1: Создайте скрипт

Создайте `EnemySpawner.cs` и повесьте его на объект `EnemySpawner`.

### Шаг 2: Добавьте код спавна

```csharp
using UnityEngine;

public class EnemySpawner : MonoBehaviour
{
    public EnemyUnit enemyPrefab;
    public Transform bottomPoint;
    public Transform topPoint;
    public GameManager gameManager;
    public float spawnInterval = 2f;

    private float nextSpawnTime;

    void Update()
    {
        if (Time.time >= nextSpawnTime)
        {
            SpawnEnemy();
            nextSpawnTime = Time.time + spawnInterval;
        }
    }

    private void SpawnEnemy()
    {
        float y = Random.Range(bottomPoint.position.y, topPoint.position.y);
        Vector3 spawnPosition = new Vector3(transform.position.x, y, 0f);

        EnemyUnit enemy = Instantiate(enemyPrefab, spawnPosition, Quaternion.identity);
        enemy.gameManager = gameManager;
    }
}
```

## Пояснение (`SpawnEnemy()`)

Метод `SpawnEnemy()` делает три вещи:

1. выбирает случайную высоту;
2. создаёт врага в этой точке;
3. передаёт врагу ссылку на `GameManager`.

Это хороший метод-действие: по имени понятно, что он делает.
А детали спрятаны внутри.

## Если враги не появляются

Проверьте:

- `EnemySpawner` стоит на сцене;
- на нём есть скрипт `EnemySpawner`;
- в поле `Enemy Prefab` лежит префаб врага, а не объект со сцены;
- `BottomPoint` и `TopPoint` назначены;
- `spawnInterval` больше `0`.

Если в Console есть `NullReferenceException`, почти всегда пустая одна из ссылок в Inspector.

## Теория 3: Что такое волна?

Бесконечный одинаковый поток врагов быстро становится скучным.

Волна — это ограниченная серия врагов.
После неё игра делает паузу, а следующая волна становится чуть сложнее.

Для простой версии достаточно:

- `enemiesPerWave` — сколько врагов в волне;
- `spawnedInWave` — сколько уже создано;
- `wavePause` — пауза между волнами.

## Пояснение (волна)

Волна — это не бесконечный поток врагов.
Это порция врагов, после которой есть короткая пауза.

Пауза нужна игроку, чтобы успеть увидеть результат, купить улучшение и приготовиться к следующей волне.

## Практика 3: Добавьте простые волны

### Шаг 1: Добавьте поля волны

Откройте `EnemySpawner.cs`.
Добавьте поля:

```csharp
public int enemiesPerWave = 3;
public float wavePause = 4f;

private int spawnedInWave;
```

### Шаг 2: Замените `Update()`

Замените `Update()` на:

```csharp
void Update()
{
    if (Time.time < nextSpawnTime)
    {
        return;
    }

    if (spawnedInWave < enemiesPerWave)
    {
        SpawnEnemy();
        spawnedInWave++;
        nextSpawnTime = Time.time + spawnInterval;
    }
    else
    {
        enemiesPerWave++;
        spawnedInWave = 0;
        nextSpawnTime = Time.time + wavePause;
        Debug.Log("Новая волна. Врагов: " + enemiesPerWave);
    }
}
```

## Как читать логику волн

> **По шагам:**
>
> 1. Если ещё не пришло время спавна, делаем `return`.
> 2. Если в волне остались враги, создаём одного врага.
> 3. Увеличиваем `spawnedInWave`.
> 4. Если волна закончилась, увеличиваем размер следующей волны.
> 5. Делаем паузу перед новой волной.

Это уже игровой цикл Tower Defence в простой форме.

## Пояснение (почему пока один тип врага)

Можно было бы добавить быстрых, сильных и редких врагов.
Но сейчас цель урока другая: понять спавнер и волну.

Один понятный враг лучше, чем три типа врагов, в которых легко запутаться.
Сложность уже растёт за счёт числа врагов в волне.

## Что проверить

Нажмите Play и посмотрите:

- враги появляются справа;
- высота появления немного разная;
- после нескольких врагов есть пауза;
- следующая волна создаёт больше врагов.

Если враги не дают монеты, проверьте ссылку `Game Manager` у `EnemySpawner`.
