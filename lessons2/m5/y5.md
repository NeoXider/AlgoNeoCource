# Модуль 5, Урок 5: Волны врагов и Баланс сложности

**Цель урока:** создать систему появления врагов (спавнер) и добавить концепцию волн, усиливая давление на игрока и усложняя игру со временем.

Механика собирается в полноценную Tower Defence. Раньше мы вручную выставляли гоблинов на сцену перед запуском игры, чтобы потестировать арбалет. Теперь игра должна бесконечно генерировать угрозы сама!

---

Если враги всегда одинаковые и идут по одному, игра перестаёт удивлять через 10 секунд.
**Волна** — это группа противников, которая появляется по заданным правилам игры. Волны задают ритм бою и позволяют наращивать сложность со временем.

---

## Теория 1: Спавнер как фабрика врагов

Все системы выстрелов в нашей игре работают одинаково. Арбалет штампует стрелы через функцию `Instantiate` по таймеру `shootCooldown`.
Спавнер врагов (генератор) работает **абсолютно по той же математике**! Но вместо стрел он создает копии гоблинов, и у него свой собственный таймер задержки.

Но спавнеру нужно не просто создавать врага, ему ещё нужно выдавать новорождённому гоблину инструкции: куда идти (передать цель), и кому отдавать золото после смерти (передать GameManager)!

### Часть 2: Простой генератор по времени

**Задача:**
1. Создайте пустой объект на правой границе экрана в самом низу поля и назовите его `Spawner`.
2. Внутри объекта `Spawner` (нажав по нему правой кнопкой мыши -> Create Empty) создайте дочерний объект `TopBound` и поднимите его на уровень верхнего края экрана.
3. Создайте скрипт `WaveSpawner.cs` и повесьте его на `Spawner`.
4. В коде добавьте ссылки на префаб гоблина, Менеджер, верхнюю границу (`topSpawnBound`), а также таймер спавна `timeBetweenSpawns`.
5. В методе `Update` сделайте таймер. Как только время вышло: рассчитайте случайную высоту от позиции Спавнера до позиции `topSpawnBound`, создайте врага через `Instantiate`, выдайте ему данные и сбросьте таймер.
6. Сохраните скрипт и вернитесь в Unity. Выделите `Spawner` и в компоненте скрипта заполните пустые поля: перетащите префаб Гоблина в `Enemy Prefab`, объект GameManager (`Manager`) в `Game Manager`, и только что созданный дочерний объект `TopBound` в поле `Top Spawn Bound`.

**Пример кода:**

```csharp
using UnityEngine;

public class WaveSpawner : MonoBehaviour
{
    public EnemyUnit enemyPrefab;
    public GameManager gameManager;
    public Transform topSpawnBound; // Верхняя граница спавна
    
    public float timeBetweenSpawns = 2f;
    private float nextSpawnTime;

    void Update()
    {
        // Если пришло время рожать нового врага
        if (Time.time >= nextSpawnTime)
        {
            // Обновляем таймер
            nextSpawnTime = Time.time + timeBetweenSpawns;
            
            // Выбираем случайную высоту от позиции Спавнера (низа) до верхней границы
            float randomY = Random.Range(transform.position.y, topSpawnBound.position.y);
            Vector3 spawnPos = new Vector3(transform.position.x, randomY, 0);

            // Создаём клона гоблина на нашей случайной высоте
            EnemyUnit newEnemy = Instantiate(enemyPrefab, spawnPos, Quaternion.identity);
            
            // Передаём ему Менеджер. Без него гоблин не даст золото.
            newEnemy.gameManager = gameManager;
        }
    }
}
```

## Теория 2: Разнообразие через Массивы

Когда на игрока идут только обычные гоблины — это скучно. Нужно добавить Орков!
Но как сказать скрипту, что у нас есть не один префаб, а несколько?

Для этого в C# существуют **Массивы** (списки). Если написать пустые квадратные скобки `[]` после типа данных, то переменная сможет хранить не один объект, а сразу целую коробку из десятка объектов!
А затем через команду `Random.Range` мы можем попросить спавнер каждый раз случайным образом доставать врага из коробки!

---

## Практика 2: Колесо фортуны

**Задача:**
1. Откройте скрипт `WaveSpawner.cs`.
2. Измените тип переменной-префаба. Вместо одного `public EnemyUnit enemyPrefab;` сделайте массив `public EnemyUnit[] enemyPrefabs;` (просто добавьте скобочки и `s` на конце).
3. При клонировании выбирайте случайный номер от 0 до количества префабов в вашей коробке: `int rand = Random.Range(0, enemyPrefabs.Length);`.
4. Заспавните вытащенного кандидата `enemyPrefabs[rand]`.
5. Вернитесь в Юнити и создайте префаб Орка (можно просто сделать копию гоблина, перекрасить его в красный цвет и задать `maxHealth = 6`). Сложите оба префаба в выпадающий массив `Enemy Prefabs` в поле Спавнера (поставив Size = 2).

**Пример измененного кода:**
```csharp
    // --- ИЗМЕНЕННОЕ ПОЛЕ ---
    public EnemyUnit[] enemyPrefabs; // Теперь это массив префабов!
    
    // ... остальной код в Start()

    void Update()
    {
        if (Time.time >= nextSpawnTime)
        {
            nextSpawnTime = Time.time + timeBetweenSpawns;
            
            // --- НОВЫЙ КОД случайного выбора ---
            int randomNum = Random.Range(0, enemyPrefabs.Length);
            
            float randomY = Random.Range(transform.position.y, topSpawnBound.position.y);
            Vector3 spawnPos = new Vector3(transform.position.x, randomY, 0);

            EnemyUnit newEnemy = Instantiate(enemyPrefabs[randomNum], spawnPos, Quaternion.identity);
            
            newEnemy.gameManager = gameManager;
        }
    }
```

## Теория 3: Волны и Баланс

Бесконечный непрерывный поток врагов — это не волны. Волны подразумевают, что враги кончаются, игроку даётся время на "выдох" и трату золота в магазине, а затем игра усложняется (сбрасывая на поле больше врагов, чем раньше).

Мы добавим счетчик врагов `enemiesPerWave`. Рождение гоблина теперь будет уменьшать этот счетчик. Как только счетчик дошел до нуля — игра ждет несколько секунд паузы, усложняет параметры будущей волны и перерождает счетчик!

### Часть 2: Совершенный Спавнер

**Задача:**
1. Добавьте в `WaveSpawner.cs` переменную размера волны `enemiesPerWave = 3`, переменную оставшихся в текущей волне `enemiesLeft`.
2. Добавьте счетчик глобальной волны `currentWaveLabel = 1` чтобы хвастаться рекордом.
3. Оберните старый спавн в условие: создаём врага **только если `enemiesLeft > 0`**. В момент спавна отнимайте `enemiesLeft--`.
4. И добавьте блок `else` "Если волна кончилась": запускаем новую волну через условные пару секунд, и повышаем сложность `enemiesPerWave += 2`.

**Пример дополненного кода:**
```csharp
    // --- НОВЫЕ ПОЛЯ ВОЛН ---
    public int enemiesPerWave = 3;
    private int enemiesLeft = 3;
    public int currentWaveLabel = 1;

    void Update()
    {
        // 1. Идет волна (враги остались)
        if (enemiesLeft > 0)
        {
            if (Time.time >= nextSpawnTime)
            {
                nextSpawnTime = Time.time + timeBetweenSpawns;
                int rand = Random.Range(0, enemyPrefabs.Length);

                float randomY = Random.Range(transform.position.y, topSpawnBound.position.y);
                Vector3 spawnPos = new Vector3(transform.position.x, randomY, 0);

                EnemyUnit newEnemy = Instantiate(enemyPrefabs[rand], spawnPos, Quaternion.identity);
                newEnemy.gameManager = gameManager;
                
                enemiesLeft--; // Уменьшаем остаток врагов для этой волны
            }
        }
        else // 2. Волна кончилась — ждем и запускаем новую, усложненную!
        {
            if (Time.time >= nextSpawnTime) // Ждем ту же самую паузу, но побольше
            {
                currentWaveLabel++;
                enemiesPerWave += 2; // УСЛОЖНЯЕМ ИГРУ: На 2 врага больше в новой волне!
                enemiesLeft = enemiesPerWave;
                
                Debug.Log("Началась волна: " + currentWaveLabel);
            }
        }
    }
```
