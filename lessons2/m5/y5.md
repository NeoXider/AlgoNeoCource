# Модуль 5, Урок 5: Волны врагов

**Цель урока:** добавить волны врагов, усилить давление на игрока и показать, как сложность может расти за счёт параметров (количество врагов, здоровье, паузы).

Механика собирается в полноценную Tower Defence: враги приходят волнами, каждая следующая может быть сложнее.

---

Если враги всегда одинаковые и идут по одному, игра быстро перестаёт удивлять.
**Волна** — это группа врагов, которая появляется по правилам. Волны задают ритм игры и позволяют постепенно увеличивать сложность.

---

## Теория 1: Что такое волна врагов

- Волна — это несколько врагов, которые появляются друг за другом (или с паузой).
- После волны можно сделать паузу, затем запустить следующую волну.
- Сложность можно повышать не только новым кодом, но и **числами**: больше врагов, больше здоровье, быстрее появление.

Даже с одним и тем же префабом врага игра сильно меняется, если менять количество и интервалы.

### Часть 2: Скрипт WaveSpawner

**Задача:**

1. Создайте пустой объект `Spawner` и поставьте его в точке, откуда должны появляться враги.
2. Создайте скрипт `WaveSpawner.cs` и повесьте на `Spawner`.
3. Добавьте поля:
   - `public EnemyUnit[] enemyPrefabs;` — массив префабов врагов (в инспекторе задайте размер и перетащите префабы). Пока можно один элемент.
   - `public Transform spawnPoint;` — точка появления (можно сам объект Spawner: `transform`).
   - `public Transform targetPoint;` — цель движения врагов (точка защиты).
   - `public GameManager gameManager;`
   - `public int enemiesPerWave = 3;` — сколько врагов в волне.
   - `public float timeBetweenSpawns = 1.2f;` — пауза между появлениями врагов в секундах.
   - Переменные для логики: сколько врагов осталось заспавнить, таймер до следующего спавна.
4. В `Update()`: если остались враги для спавна и таймер истёк — создайте врага через `Instantiate`, задайте ему `targetPoint` и `gameManager`, уменьшите счётчик и сбросьте таймер. Когда волна закончилась — через некоторое время начните новую (например, увеличьте количество врагов или здоровье).

**Упрощённый пример логики:**

- Храните `int enemiesLeftToSpawn` и `float spawnTimer`.
- Каждый кадр уменьшайте `spawnTimer` на `Time.deltaTime`.
- Когда `spawnTimer <= 0` и `enemiesLeftToSpawn > 0`: вызовите метод спавна одного врага, уменьшите `enemiesLeftToSpawn`, установите `spawnTimer = timeBetweenSpawns`.
- Для следующей волны: когда `enemiesLeftToSpawn == 0`, заведите таймер паузы; по истечении увеличьте номер волны и снова задайте `enemiesLeftToSpawn` (например, `enemiesPerWave + номер_волны`).

**Пример спавна одного врага:**

```csharp
void SpawnEnemy()
{
    if (enemyPrefabs == null || enemyPrefabs.Length == 0) return;
    EnemyUnit prefab = enemyPrefabs[0];
    EnemyUnit enemy = Instantiate(prefab, spawnPoint.position, spawnPoint.rotation);
    enemy.targetPoint = targetPoint;
    enemy.gameManager = gameManager;
}
```

## Теория 2: Параметры сложности

Простые параметры волны:

- **Количество врагов** в волне.
- **Пауза между появлениями** — чем меньше, тем плотнее волна.
- **Здоровье врага** — можно увеличивать с номером волны (например, `maxHealth + currentWave`).
- **Скорость врага** — по желанию.

Один и тот же скрипт врага может работать с разными числами: разные префабы с разными настройками или одни и те же префабы с изменёнными в коде параметрами после `Instantiate`.

### Часть 2: Первая и вторая волны

**Задача:**

1. Реализуйте запуск первой волны в `Start()`: задайте `enemiesLeftToSpawn = enemiesPerWave` (или аналогично).
2. Когда все враги волны заспавнены, подождите несколько секунд (таймер между волнами), затем увеличьте номер волны и снова задайте количество врагов (например, `enemiesPerWave + currentWave - 1`).
3. При спавне врага можно увеличивать ему здоровье в зависимости от номера волны: `enemy.maxHealth += currentWave - 1; enemy.currentHealth = enemy.maxHealth;` (после Instantiate).
4. Проверьте в игре: идёт первая волна, после паузы — вторая, враги становятся живучее или их больше.

## Как показать разных врагов без перегруза?

Полную сложную систему типов врагов давать рано. Достаточно показать, что **разница часто делается префабами и числами**: один префаб — обычный враг, другой — с большим здоровьем или скоростью. Один и тот же скрипт `EnemyUnit` работает с разными настройками.

---

## Теория 3: Разные враги через префабы и параметры

- Один враг может быть быстрее, другой — живучее. В скрипте уже есть `moveSpeed`, `maxHealth` — их можно задавать в префабе по-разному.
- Массив префабов `enemyPrefabs[]` позволяет выбирать, кого спавнить: например, по номеру волны или случайно `Random.Range(0, enemyPrefabs.Length)`.
- Баланс строится на параметрах и настройках, а не обязательно на сложной архитектуре.

## Практика 3: Несколько типов врагов (по желанию)

**Задача:**

1. Создайте второй префаб врага (скопируйте первый и измените, например, `maxHealth` или `moveSpeed`).
2. В `WaveSpawner` в массиве `enemyPrefabs` укажите два (или больше) префаба.
3. В методе спавна выбирайте префаб: например, всегда первый в первой волне; со второй волны — случайный из массива: `int index = Random.Range(0, enemyPrefabs.Length);` и спавньте `enemyPrefabs[index]`.
4. Проверьте: в разных волнах появляются разные по силе враги, игрок использует улучшения.
