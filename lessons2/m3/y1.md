## Как заставить Dino прыгать только “ЕСЛИ”?

В играх почти всё строится на проверках: **ЕСЛИ** произошло событие — делаем действие.

Например, мы не хотим, чтобы динозавр прыгал каждый кадр. Мы хотим, чтобы он прыгал **когда нужно**:

- игрок нажал `Space`
- динозавр стоит на земле
- игра ещё не окончена

Чтобы такие проверки работали, нам нужен тип данных **`bool`** (читается “булевый”):

- `bool` хранит только два значения: `true` (да) или `false` (нет)
- это как переключатель: “можно/нельзя”, “есть/нет”, “нажат/не нажат”

Сегодня мы научимся получать `bool` из сравнений и писать условия `if`, которые превращают “хочу так” в работающую игровую логику.

## Теория 1: Операции сравнения (bool)

Операции сравнения возвращают (дают нам) `bool` — `true` или `false`.

Основные операторы:

- `==` равенство
- `!=` не равно
- `<` меньше
- `>` больше
- `<=` меньше или равно
- `>=` больше или равно

Примеры для каждого оператора:

```csharp
int a = 10;
int b = 4;

bool eq = a == b;   // false
bool ne = a != b;   // true
bool lt = a < b;    // false
bool gt = a > b;    // true
bool le = a <= b;   // false
bool ge = a >= b;   // true
```

Важно: операторы `< > <= >=` имеют более высокий приоритет, чем `==` и `!=`.

## Теория 2: if / else — базовая игровая логика

`if` проверяет условие. Если оно `true`, мы выполняем действие.

Если условие `false`, можно выполнить “иначе” через `else`.

Пример:

```csharp
int coins = 3;
int price = 5;

if (coins >= price)
    Debug.Log("Покупка успешна!");
else
    Debug.Log("Не хватает монет");
```

## Теория 3: Ввод игрока — `Input`

Unity даёт нам класс `Input`, чтобы проверять действия игрока.

Самые полезные варианты для клавиш:

- `Input.GetKeyDown(...)` — **сработает один раз** в кадре, когда кнопку **нажали**
- `Input.GetKey(...)` — будет `true`, **пока кнопку держат**
- `Input.GetKeyUp(...)` — сработает один раз в кадре, когда кнопку **отпустили**

Почему это обычно пишут в `Update()`:

- `Update()` вызывается каждый кадр → мы регулярно “слушаем” ввод
- если проверять ввод в `Start()` — он проверится только один раз в начале игры

Пример:

```csharp
if (Input.GetKeyDown(KeyCode.Space))
    Debug.Log("Нажали Space (один раз)");
```

Прежде чем делать прыжок, разберём ещё несколько новых вещей:

- `Rigidbody` — компонент физики, который помогает объекту двигаться “по-настоящему”;
- `GetComponent<Rigidbody>()` — способ найти `Rigidbody` на этом же объекте;
- `new Vector3(0, jumpForce, 0)` — новый вектор, где мы задаём движение вверх по оси `Y`;
- `rb.velocity` — текущая скорость объекта.

Если у вас Unity 2022, обычно для `Rigidbody` используют именно `velocity`.
В более новых версиях Unity вы можете встретить и `linearVelocity`, но смысл для нас сейчас один и тот же: это скорость объекта.

В этом уроке нам достаточно простой идеи:
мы берём `Rigidbody` и задаём ему скорость вверх, чтобы получился прыжок.

### Шаг 3: Вызывайте `Jump()` по кнопке

Теперь в `Update()` проверьте `Input.GetKeyDown(KeyCode.Space)` и при нажатии вызывайте `Jump()`.

```csharp
using UnityEngine;

public class DinoController : MonoBehaviour
{
    public float speed = 5f;
    public float jumpForce = 5f;
    private Rigidbody rb;

    void Awake()
    {
        rb = GetComponent<Rigidbody>();
    }

    void Update()
    {
        transform.position += Vector3.right * speed * Time.deltaTime;

        if (Input.GetKeyDown(KeyCode.Space))
            Jump();
    }

    void Jump()
    {
        rb.velocity = new Vector3(0, jumpForce, 0);
    }
}
```

## Важно: как обычно делают “бесконечные” раннеры
