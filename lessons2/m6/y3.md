# Модуль 6, Урок 3: GameManager и очки

**Цель урока:** ввести общий менеджер игры (`GameManager`), счёт очков, состояние «игра окончена» и простой UI. В конце добавим небольшое ускорение темпа.

---

Игрок, враги и астероиды делают своё дело. Но кто считает очки, решает, что игра закончилась, и обновляет надпись на экране? Для этого нужен один общий объект — **GameManager**.

---

## Теория 1: Роль GameManager

**GameManager** — скрипт на одном объекте сцены, который хранит общее состояние игры:

- счёт (очки);
- флаг «игра окончена» (`IsGameOver`);
- при необходимости — множитель сложности (скорость врагов/астероидов).

Важно не менять эти данные отовсюду напрямую. Лучше:

- увеличивать счёт через метод `AddScore(int points)`;
- завершать игру через метод `FinishGame()`.

Так все изменения проходят через один «контролёр», код остаётся предсказуемым и аккуратным.

Иногда такие данные удобно прятать за **свойствами C#**. Свойство — это как «умная переменная», где вы сами решаете, что произойдёт при получении или изменении значения. Такое поле называется **свойство**.

```csharp
public int Score { get; private set; }
public bool IsGameOver { get; private set; }
```

- `get` — как «взять значение» (читать можно в любом скрипте);
- `private set` — как «установить значение», но только внутри самого GameManager.

Так другие скрипты смогут читать `Score` и `IsGameOver`, но менять их смогут только методы самого менеджера (`AddScore`, `FinishGame`).

> **Пояснение:**  
> - Свойство работает примерно как обычная переменная, но через него можно добавить свою логику при чтении или записи значения: например, автоматически вызывать дополнительные методы или проверки.
> - Это инструмент для более «контролируемого» доступа к данным.
> - Учтите, что такие свойства **не отображаются в инспекторе Unity**! Для вывода значения в редакторе используйте обычные поля с атрибутом `[SerializeField]` или `public` переменные.

## Теория 2: UI должен узнавать об изменениях

Чтобы UI (текст очков, панель Game Over) обновлялся при изменении счёта или при проигрыше, нужен понятный путь связи.

В больших проектах часто используют события: `GameManager` сообщает «счёт изменился», а UI реагирует.
Но для нашей версии шутера достаточно проще:

- `GameManager` хранит ссылку на `scoreText`;
- когда счёт меняется, вызывает `UpdateScoreText()`;
- когда игра закончилась, включает `gameOverPanel`.

```csharp
public void AddScore(int points)
{
    if (IsGameOver)
    {
        return;
    }

    Score += points;
    UpdateScoreText();
}
```

> **Как работает связка GameManager и UI по шагам:**
>
> 1. `AddScore(points)` увеличивает внутренний счёт.
> 2. `UpdateScoreText()` обновляет текст очков.
> 3. `FinishGame()` включает панель Game Over.
> 4. Другие скрипты не меняют UI напрямую — они только вызывают методы `GameManager`.

## Практика 1: GameManager и счёт

**Задача:**

### Шаг 1: Создайте GameManager

1. Создайте пустой объект `GameController` (или `GameManager`), добавьте скрипт `GameManager.cs`.

### Шаг 2: Добавьте свойства состояния

Добавьте поля или свойства для счёта и флага конца игры. Удобно сразу сделать их **свойствами только для чтения снаружи**:

```csharp
public int Score { get; private set; }
public bool IsGameOver { get; private set; }
```

Счёт лучше увеличивать только в методе `AddScore(int points)` и только если `!IsGameOver`. Внутри GameManager можно менять свойства через `Score += points;` и `IsGameOver = true;`.

### Шаг 3: Подключите счёт к UI

Выведите счёт на UI через `TMP_Text`: обновляйте текст внутри `AddScore`.

**Простой пример GameManager:**

```csharp
using TMPro;
using UnityEngine;

public class GameManager : MonoBehaviour
{
    [SerializeField] private TMP_Text scoreText;
    [SerializeField] private GameObject gameOverPanel;

    public int Score { get; private set; }
    public bool IsGameOver { get; private set; }

    private void Start()
    {
        gameOverPanel.SetActive(false);
        UpdateScoreText();
    }

    public void AddScore(int points)
    {
        if (IsGameOver)
        {
            return;
        }

        Score += points;
        UpdateScoreText();
    }

    public void FinishGame()
    {
        if (IsGameOver)
        {
            return;
        }

        IsGameOver = true;
        gameOverPanel.SetActive(true);
    }

    private void UpdateScoreText()
    {
        scoreText.text = "Очки: " + Score;
    }
}
```

### Шаг 4: Начислите очки за уничтожение

При уничтожении астероида или врага вызовите `AddScore(10)` у `GameManager`.
Проще всего передать ссылку на `GameManager` через инспектор или получить её заранее в `Start()`.

---

## Если появилась `NullReferenceException`

Чаще всего здесь ошибка будет в строке:

```csharp
scoreText.text = "Очки: " + Score;
```

Проблема не в строке `"Очки: "`.
Проблема перед точкой: `scoreText` не указывает на TMP-текст.

Проверьте в Inspector:

- у `GameManager` заполнено поле `Score Text`;
- в поле перетащен именно объект с компонентом `TMP_Text`;
- `Game Over Panel` тоже назначен.

## Теория 3: Рост сложности

Сложность можно поднимать не новыми врагами, а изменением уже существующих параметров:

- увеличивать скорость астероидов и врагов со временем;
- или уменьшать интервал спавна.

Для этого в GameManager хранят, например, `speedMultiplier`. Раз в N секунд его увеличивают (`InvokeRepeating(nameof(IncreaseDifficulty), 10f, 10f)`). Астероиды и враги при движении умножают свою базовую скорость на `gameManager.SpeedMultiplier`. Так темп постепенно растёт.

---

> **Как работает постепенное усложнение через InvokeRepeating:**
>
> 1. В `Start()` GameManager один раз вызывается `InvokeRepeating(nameof(IncreaseDifficulty), 10f, 10f)`.
> 2. Unity ждёт 10 секунд, затем вызывает `IncreaseDifficulty()` и повторяет это каждые 10 секунд.
> 3. Внутри `IncreaseDifficulty` вы увеличиваете `speedMultiplier` (например, `speedMultiplier += 0.2f;`).
> 4. Скрипты движения берут скорость как `baseSpeed * speedMultiplier`, поэтому с каждым вызовом метода игра становится чуть быстрее.

В скриптах движения астероидов и врагов используйте скорость с учётом множителя (например, брать `baseSpeed * gameManager.SpeedMultiplier`). Не забудьте проверять `!IsGameOver` перед увеличением сложности в GameManager.
