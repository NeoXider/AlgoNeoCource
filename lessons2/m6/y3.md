# Модуль 6, Урок 3: GameManager и очки

**Цель урока:** ввести общий менеджер игры (GameManager), счёт очков, состояние «игра окончена» и связь с UI через события. Добавить постепенное ускорение темпа.

---

Игрок, враги и астероиды делают своё дело. Но кто считает очки, решает, что игра закончилась, и обновляет надпись на экране? Для этого нужен один общий объект — **GameManager**.

---

## Теория 1: Роль GameManager

**GameManager** — скрипт на одном объекте сцены, который хранит общее состояние игры:

- счёт (очки);
- флаг «игра окончена» (`isGameOver`);
- при необходимости — множитель сложности (скорость врагов/астероидов).

Важно не менять эти данные отовсюду напрямую. Лучше:

- увеличивать счёт через метод `AddScore(int points)`;
- завершать игру через метод `FinishGame()`.

Так все изменения проходят через один «контролёр», код остаётся предсказуемым и аккуратным.

## Теория 2: События и UI

Чтобы UI (текст очков, панель Game Over) обновлялся при изменении счёта или при проигрыше, не обязательно давать UI прямую ссылку на переменную счёта. Удобнее использовать **события**:

- При вызове `AddScore` GameManager вызывает событие «счёт изменился» и передаёт новое значение.
- UI подписывается на это событие и обновляет надпись.
- При вызове `FinishGame` GameManager вызывает событие «игра окончена»; UI подписывается и показывает панель Game Over.

В Unity для этого используют `UnityEvent`. Если нужно передать число (новый счёт), делают класс-обёртку: `UnityEvent<int>`. В коде это выглядит так:

```csharp
[Serializable]
public class ScoreChangedEvent : UnityEvent<int> { }

[SerializeField] private ScoreChangedEvent onScoreChanged = new ScoreChangedEvent();

public void AddScore(int points)
{
    if (!isGameOver)
    {
        score += points;
        onScoreChanged.Invoke(score);
    }
}
```

UI в `Start()` подписывается: `gameManager.OnScoreChanged.AddListener(UpdateScoreText);`, а метод `UpdateScoreText(int value)` выводит значение в текст.

## Практика 1: GameManager и счёт

**Задача:**

1. Создайте пустой объект `GameController` (или `GameManager`), добавьте скрипт `GameManager.cs`.
2. Добавьте поля: счёт (int), isGameOver (bool). Счёт лучше увеличивать только в методе `AddScore(int points)` и только если `!isGameOver`.
3. При уничтожении астероида или врага вызовите `AddScore(10)` у `GameManager`. Проще всего передать ссылку на `GameManager` через инспектор или получить её заранее в `Start()`.
4. Выведите счёт на UI: можно сразу подписать `GameHud` на событие изменения счёта или временно обновлять текст внутри `AddScore`.

**Упрощённый пример GameManager (без событий, только счёт и конец игры):**

```csharp
[SerializeField] private int score;
[SerializeField] private bool isGameOver;
[SerializeField] private TMP_Text scoreText; // ссылка из инспектора

public void AddScore(int points)
{
    if (!isGameOver)
    {
        score += points;
        scoreText.text = "Очки: " + score;
    }
}

public void FinishGame()
{
    if (!isGameOver)
    {
        isGameOver = true;
        // показать панель Game Over
    }
}
```

## Теория 3: Рост сложности

Сложность можно поднимать не новыми врагами, а изменением уже существующих параметров:

- увеличивать скорость астероидов и врагов со временем;
- или уменьшать интервал спавна.

Для этого в GameManager хранят, например, `speedMultiplier`. Раз в N секунд его увеличивают (`InvokeRepeating(nameof(IncreaseDifficulty), 10f, 10f)`). Астероиды и враги при движении умножают свою базовую скорость на `gameManager.SpeedMultiplier`. Так темп постепенно растёт.

## Практика 3: Постепенное ускорение

**Задача:**

1. В GameManager добавьте `speedMultiplier` (float, начальное значение 1f) и метод `IncreaseDifficulty()`, который увеличивает его на небольшой шаг (например, на 0.2f).
2. В `Start()` GameManager вызовите `InvokeRepeating(nameof(IncreaseDifficulty), 10f, 10f)` — раз в 10 секунд сложность растёт.
3. В скриптах движения астероидов и врагов используйте скорость с учётом множителя (например, брать `baseSpeed * gameManager.SpeedMultiplier`). Не забудьте проверять `!isGameOver` перед вызовом IncreaseDifficulty в GameManager.
