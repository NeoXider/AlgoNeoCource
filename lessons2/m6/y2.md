# Модуль 6, Урок 2: Invoke и спавнеры

**Цель урока:** научиться вызывать методы по таймеру через `InvokeRepeating`, добавить спавнер астероидов и простых врагов, которые летят вниз и стреляют.

---

Когда объекты появляются сами по времени, игра живёт без постоянного участия игрока. Для этого нужны таймеры и спавнеры.

---

## Теория 1: Вызов метода по времени — InvokeRepeating

Unity позволяет вызывать метод не сразу, а через паузу, и повторять вызов через заданный интервал.

```csharp
InvokeRepeating("ИмяМетода", задержкаДоПервогоВызова, интервалПовторения);
```

Пример: вызывать метод `SpawnAsteroid` первый раз через 1 секунду, потом каждые 1,5 секунды:

```csharp
void Start()
{
    InvokeRepeating(nameof(SpawnAsteroid), 1f, 1.5f);
}

void SpawnAsteroid()
{
    // создаём астероид
}
```

`nameof(SpawnAsteroid)` — это имя метода в виде строки, так удобнее и безопаснее, чем писать `"SpawnAsteroid"` вручную.

## Практика 1: Стрельба по времени (опционально)

Можно перенести выстрел игрока с «по нажатию» на «каждые N секунд» через `InvokeRepeating` в `Start()`. Это не обязательно для шутера с ручным выстрелом, но полезно для понимания таймера.

## Теория 2: Спавнер и случайная позиция

**Спавнер** — объект, который создаёт другие объекты (астероиды, врагов) по таймеру.

Чтобы объекты появлялись в разных местах по горизонтали, используют случайное число:

```csharp
float randomX = Random.Range(minX, maxX);
Vector3 position = new Vector3(randomX, spawnY, 0f);
Instantiate(prefab, position, Quaternion.identity);
```

`Random.Range(minX, maxX)` даёт случайное число в диапазоне от minX до maxX.

Атрибуты Unity помогают сделать инспектор понятнее:

```csharp
[Header("Спавн астероидов")]
[SerializeField] private float asteroidRate = 1.5f;
```

`[Header("...")]` — рисует подзаголовок в инспекторе над полем.

## Практика 2: Спавнер астероидов

**Задача:**

1. Создайте объект астероида (спрайт или примитив), добавьте движение вниз и при выходе за экран — `Destroy(gameObject)`. Сделайте префаб.
2. Создайте пустой объект `Spawner`, добавьте скрипт `TimedSpawner.cs`.
3. В `Start()` вызовите `InvokeRepeating(nameof(SpawnAsteroid), startDelay, asteroidRate)`.
4. В методе `SpawnAsteroid()` вычислите позицию через `Random.Range(minX, maxX)` и `Instantiate(asteroidPrefab, position, Quaternion.identity)`.
5. Назначьте префаб астероида в инспекторе.

**Пример TimedSpawner (только астероиды):**

```csharp
[Header("Астероиды")]
[SerializeField] private GameObject asteroidPrefab;
[SerializeField] private float minX = -7f, maxX = 7f, spawnY = 6f;
[SerializeField] private float asteroidStartDelay = 1f, asteroidRate = 1.5f;

void Start()
{
    InvokeRepeating(nameof(SpawnAsteroid), asteroidStartDelay, asteroidRate);
}

void SpawnAsteroid()
{
    float randomX = Random.Range(minX, maxX);
    Vector3 position = new Vector3(randomX, spawnY, 0f);
    Instantiate(asteroidPrefab, position, Quaternion.identity);
}
```

## Теория 3: Враг с движением вниз и выстрелом

Враг может:

- лететь вниз (как астероид);
- стрелять по таймеру через `InvokeRepeating(nameof(Fire), startDelay, fireRate)`.

Пуля врага летит **вниз** (`Vector3.down`) и тоже удаляется через время.

Если у астероида и врага общая логика «двигаться вниз и уничтожиться за экраном», её можно вынести в **базовый класс**. Дочерний класс тогда наследует движение и добавляет своё (например, выстрел или реакцию на попадание). Это тема ООП из модуля 5: один скрипт — общая часть, другой — дополнение.

## Практика 3: Астероид с уничтожением от пули

**Задача:**

1. На префаб астероида добавьте Collider (2D) и включите Is Trigger.
2. В скрипте астероида в `OnTriggerEnter2D(Collider2D other)` проверьте, есть ли у `other` компонент пули игрока (`GetComponent<ProjectileMover>()`). Если да — уничтожьте и пулю, и астероид: `Destroy(other.gameObject); Destroy(gameObject);`

```csharp
void OnTriggerEnter2D(Collider2D other)
{
    if (other.GetComponent<ProjectileMover>() != null)
    {
        Destroy(other.gameObject);
        Destroy(gameObject);
    }
}
```
