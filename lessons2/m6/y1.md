---

В шутере главное — чтобы игрок сразу почувствовал управление и выстрел. Если корабль двигается и стреляет, игра уже ощущается.

## Теория 1: Движение по осям и ограничение позиции

В космическом шутере корабль обычно двигается влево-вправо и вверх-вниз в пределах экрана.

Для плавного ввода по осям в Unity используют:

```csharp
float moveX = Input.GetAxis("Horizontal"); // -1..0..1 (стрелки или A/D)
float moveY = Input.GetAxis("Vertical");   // -1..0..1 (стрелки или W/S)
```

Чтобы корабль не улетал за экран, позицию ограничивают:

```csharp
position.x = Mathf.Clamp(position.x, minX, maxX);
position.y = Mathf.Clamp(position.y, minY, maxY);
```

`Mathf.Clamp(значение, минимум, максимум)` не даёт значению выйти за границы.

## Теория 2: Снаряд и выстрел

Выстрел — это создание объекта снаряда в точке дула корабля.

- `Instantiate(префаб, позиция, поворот)` — создаёт копию объекта на сцене.
- Пуля должна лететь вверх и **исчезать через время**, иначе сцена заполнится объектами.
- `Destroy(gameObject, 2f)` — удалит объект через 2 секунды.
- Чтобы выстрел срабатывал **один раз по нажатию**, удобно использовать `Input.GetKeyDown(KeyCode.Space)`.

Пример движения пули вверх:

```csharp
transform.position += Vector3.up * speed * Time.deltaTime;
```

Пример проверки нажатия:

```csharp
if (Input.GetKeyDown(KeyCode.Space))
{
    Instantiate(projectilePrefab, firePoint.position, firePoint.rotation);
}
```

## Практика 2: Управление кораблём

**Задача:**

1. Создайте скрипт `PlayerShipController.cs` и повесьте на корабль.
2. Добавьте переменные: скорость движения, границы экрана (`minX`, `maxX`, `minY`, `maxY`).
3. В `Update()` читайте `Input.GetAxis("Horizontal")` и `Input.GetAxis("Vertical")`, двигайте `transform.position` и ограничивайте его через `Mathf.Clamp`.

**Пример кода движения:**

```csharp
void Update()
{
    float moveX = Input.GetAxis("Horizontal");
    float moveY = Input.GetAxis("Vertical");

    Vector3 position = transform.position;
    position.x += moveX * moveSpeed * Time.deltaTime;
    position.y += moveY * moveSpeed * Time.deltaTime;
    position.x = Mathf.Clamp(position.x, minX, maxX);
    position.y = Mathf.Clamp(position.y, minY, maxY);
    transform.position = position;
}
```

## Теория 3: Поля для настройки — public и [SerializeField]

В Unity поля с модификатором `public` видны в инспекторе — их удобно настраивать. Но не все данные нужно «открывать» всему коду.

- **public** — поле видно в инспекторе и из других скриптов.
- **private** — поле видно только внутри своего класса, в инспекторе по умолчанию не отображается.
- **[SerializeField] private** — поле остаётся приватным для кода, но **отображается в инспекторе**. Так можно настраивать скорость и границы, не делая их доступными отовсюду.

Пример:

```csharp
[SerializeField] private float moveSpeed = 5f;
[SerializeField] private float minX = -7f, maxX = 7f;
```

Так код становится аккуратнее: настройки в инспекторе есть, но менять их из других скриптов не обязательно.
