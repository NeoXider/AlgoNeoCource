# Модуль 2, Урок 5: Жизненный цикл Unity + эффекты и звук в кликере

---

Вы наверняка сталкивались с “магией Unity”: почему ссылка `null`, почему звук не играет, почему физика дёргается?
Чтобы это перестало быть магией, нужно понять, **когда Unity вызывает ваш код**.

---

## Теория 1: Зачем нужен жизненный цикл

В Unity ваш скрипт живёт “внутри” объекта и получает вызовы от движка в определённом порядке. Это и есть **жизненный цикл**.

Почему это важно:

- вы понимаете, **когда** и **почему** запускается ваш код
- вы не ловите “магические” баги вроде: “почему ссылка пустая?” или “почему физика дёргается?”

---

Если знать примерный порядок вызовов, легко выбрать “правильное место” для кода: где искать компоненты, где читать ввод, а где работать с физикой.

## Теория 2: Краткая карта жизненного цикла Unity (очень упрощённо)

Порядок (в реальной игре есть нюансы, но для нас этого достаточно):

1. **Создание/загрузка объекта**

   - `Awake()` — один раз, объект появился
   - `OnEnable()` — каждый раз, когда объект включили (`enabled = true` или объект активен)

1. **Старт игры**

   - `Start()` — один раз, перед первым кадром (после `Awake/OnEnable`)

1. **Игровой цикл (пока идёт Play Mode)**

   - `Update()` — каждый кадр (ввод, UI, “обычная” логика)
   - `FixedUpdate()` — по фиксированному шагу (физика)
   - `LateUpdate()` — каждый кадр, но после `Update()` (камера/догонялки)

1. **Отключение/удаление**

   - `OnDisable()` — когда объект выключили
   - `OnDestroy()` — когда объект уничтожили

---

Теперь разберём основные методы подробно — именно они чаще всего встречаются в проектах: `Awake`, `Start`, `Update`, `FixedUpdate`, `LateUpdate`.

## Теория 3: Основные методы — погружаясь в детали

### `Awake()`

Вызывается **один раз**, когда объект создаётся/загружается.

Обычно сюда кладут:

- поиск и кеширование компонентов (`GetComponent`)
- начальную инициализацию данных

Пример:

Этот код находит и сохраняет ссылку на `AudioSource` один раз при загрузке объекта, чтобы потом быстро проигрывать звук.

```csharp
private AudioSource audioSource;

private void Awake()
{
    audioSource = GetComponent<AudioSource>();
}
```

---

### `OnEnable()`

Вызывается **каждый раз**, когда объект/компонент становится активным (включили компонент или активировали объект).

Обычно сюда кладут:

- подписку на события (`AddListener`, `+=`)
- запуск корутин/таймеров, которые должны работать только пока объект “включён”

Пример:

```csharp
private void OnEnable()
{
    Debug.Log("Я включился!");
}
```

---

### `Start()`

Вызывается **один раз**, перед первым кадром игры.

Обычно сюда кладут:

- стартовые настройки (например, показать стартовый счёт)
- запуск таймеров/анимаций

Пример:

```csharp
private void Start()
{
    Debug.Log("Игра началась!");
}
```

---

### `Update()`

Вызывается **каждый кадр**.

Обычно сюда кладут:

- то, что должно постоянно обновляться
- работу со временем через `Time.deltaTime`

Важно: если в `Update()` делать “тяжёлую” логику, игра начнёт тормозить.

---

### `FixedUpdate()`

Вызывается **через равные промежутки времени**, а не “по кадрам” (физический шаг, по умолчанию 0.02 сек).

Когда использовать:

- если вы двигаете объект через **Rigidbody/Rigidbody2D** и хотите стабильную физику

Мини‑пример:

Этот код каждый физический шаг “толкает” объект вперёд через Rigidbody (движение будет стабильным).

```csharp
public Rigidbody rb;
public float force = 5f;

private void FixedUpdate()
{
    rb.AddForce(Vector3.forward * force);
}
```

---

### `LateUpdate()`

Вызывается **каждый кадр, но после `Update()`**.

Когда использовать:

- если нужно “догнать” объект после того, как он уже сдвинулся в `Update()` (например, камера следует за игроком)

Мини‑пример:

Этот код делает простое “следование камеры”: каждый кадр после движения цели камера ставит позицию относительно `target`.

```csharp
public Transform target;

private void LateUpdate()
{
    transform.position = target.position + new Vector3(0, 5, -8);
}
```

## Теория 4: Остальные методы

- `OnDisable()` — вызывается, когда объект/компонент выключили; обычно тут **отписываются от событий** и останавливают процессы. Пример: “снять подписку `button.onClick.RemoveListener(...)`”.
- `OnDestroy()` — вызывается перед уничтожением объекта; используйте для финальной очистки (если объект удаляется). Пример: “закрыть соединение/остановить звук/удалить таймер”.
- `Awake()` vs `Start()` — `Awake()` про “подготовить ссылки”, `Start()` про “запустить игру, когда сцена готова”. Пример: `Awake(): audioSource = GetComponent<AudioSource>(); Start(): UpdateUI();`.
- `Reset()` — вызывается в редакторе при добавлении компонента; удобно выставить “значения по умолчанию”. Пример: “сразу поставить `clickPower = 1`”.
- `OnValidate()` — вызывается в редакторе при изменении полей в инспекторе; удобно автоматически исправлять неверные значения. Пример: “если `clickPower < 1`, сделать 1”.
- `LateUpdate()` — хорошо для камеры и визуального “догоняющего” кода. Пример: “камера всегда после движения игрока”.
- `FixedUpdate()` — хорошо для физики и Rigidbody‑движения. Пример: “AddForce/MovePosition в FixedUpdate”.

### 💡 Лайфхак: `GetComponent` — “волшебный” способ найти компонент

`GetComponent<Тип>()` — это метод Unity, который **ищет компонент на этом же объекте** и возвращает ссылку на него.

Когда он нужен:

- когда вам нужно из кода управлять компонентом (например, `AudioSource`, `Rigidbody`, `Animator`, `SpriteRenderer`)
- когда вы хотите “подключиться” к уже добавленному компоненту, а не хранить всё руками в инспекторе

Мини‑пример:

Этот код находит `AudioSource` на текущем объекте и возвращает ссылку на него.

```csharp
AudioSource audioSource = GetComponent<AudioSource>();
```

---

### 💡 Лайфхак: почему ссылку лучше “кешировать” в `Awake()`, а не искать в `Start()`

Проблема не в том, что `GetComponent` в `Start()` “сломает игру” — обычно он работает нормально.  
Но **в `Awake()` лучше по привычке**, потому что:

- `Awake()` вызывается раньше `Start()` → ссылка будет готова ещё до того, как другие скрипты начнут работать в `Start()`
- вы отделяете “поиск ссылок” (подготовка) от “логики старта игры”
- самое важное: **нельзя делать `GetComponent` каждый кадр** (например, в `Update()`), это лишняя работа (ПОТЕРЯ FPS)

## Пояснение (Awake)

- `Start()` часто тоже подходит, но `Awake()` вызывается раньше — поэтому это “безопасная привычка”, если ссылка нужна уже на старте.
- Обычно в `Start()` делают начальную настройку и запускают стартовую логику игры (показать UI, выставить начальные значения, запустить таймер/корутину).

- `Awake()` вызывается один раз при создании/загрузке объекта (раньше `Start()`), и там часто подготавливают ссылки (`GetComponent`, кеширование).

### 💡 Лайфхак: Spawn через `Instantiate` и удаление через `Destroy`

**`Instantiate`** — создаёт копию объекта (префаба) в сцене.

Сигнатура, которую мы будем использовать чаще всего:

```csharp
Instantiate(prefab, position, rotation);
```

Ещё две полезные перегрузки с родителем:

```csharp
Instantiate(prefab, parent);
```

```csharp
Instantiate(prefab, position, rotation, parent);
```

Пояснение параметров:

- `prefab` — префаб, который хотим создать (например, `clickFxPrefab`)
- `position` — позиция в мире, где появится объект
- `rotation` — поворот объекта (часто `Quaternion.identity`, если не нужен)
- `parent` — Transform родителя, чтобы объект сразу появился в нужной иерархии (удобно для UI)

Пример:

```csharp
ParticleSystem fx = Instantiate(clickFxPrefab, transform.position, Quaternion.identity);
```

**`Destroy`** — удаляет объект из сцены.

>Важно: удалять нужно **сам игровой объект** (через `gameObject`), иначе вы удалите только компонент.

Две основные перегрузки:

```csharp
Destroy(obj);
```

```csharp
Destroy(obj, delay);
```

Где `obj` — объект для удаления (например, `fx.gameObject`), `delay` — задержка в секундах.

Удобно удалять эффекты через время:

```csharp
Destroy(fx.gameObject, lifeTime);
```

Где `lifeTime` — сколько секунд эффект живёт (например, `duration + startLifetime`).

Пример (ориентир):

Этот скрипт добавляет “обратную связь” на клик: при нажатии увеличивает счёт и запускает звук + частицы.

```csharp
using TMPro;
using UnityEngine;
using UnityEngine.EventSystems;

public class ClickerGame : MonoBehaviour, IPointerDownHandler
{
    public float score = 0;
    public int clickPower = 1;
    public int penaltyAmount = 5;
    public TMP_Text textScore;

    public AudioClip clickClip;
    public ParticleSystem clickFxPrefab;

    private AudioSource clickAudioSource;

    private void Awake()
    {
        clickAudioSource = GetComponent<AudioSource>();
    }

    private void Update()
    {
        score -= penaltyAmount * Time.deltaTime;
        transform.localScale = Vector3.one * (1 + score / 100f);
        textScore.text = ((int)score).ToString();
    }

    public void OnPointerDown(PointerEventData eventData)
    {
        score += clickPower;
        PlayClickFeedback();
    }

    private void PlayClickFeedback()
    {
        if (clickAudioSource != null && clickClip != null)
            clickAudioSource.PlayOneShot(clickClip);

        if (clickFxPrefab != null)
        {
            ParticleSystem fx = Instantiate(clickFxPrefab, transform.position, Quaternion.identity);
            fx.Play();
            Destroy(fx.gameObject, 1f);
        }
    }
}
```

## Пояснение: AudioSource берется автоматически

`AudioSource` мы получаем через `GetComponent<AudioSource>()` в `Awake()`.
