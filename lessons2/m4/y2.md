# Модуль 4, Урок 2: Крестики-нолики — очередь хода и логика клетки

**Цель урока:** сделать настоящую очередность `X` и `O`, запретить повторный клик по занятой клетке и научить игру аккуратно обрабатывать ход игрока.

На прошлом уроке игра уже умела показывать `X` после клика.
Теперь мы превращаем этот первый клик в настоящее правило игры: у хода появляется порядок, а у клетки появляется состояние.

---

## Как игре понять, чей сейчас ход?

На прошлом уроке по клетке можно было нажать, и в ней появлялся символ.
Теперь игра должна стать умнее:

- понимать, чей сейчас ход;
- ставить разные символы;
- не разрешать нажимать в занятую клетку.

---

Когда появляются правила игры, одного клика уже недостаточно.
Теперь нужно, чтобы код принимал решения по состоянию поля.

---

## Как отличить пустую клетку от занятой?

Если игрок нажал на клетку, игра должна быстро ответить:

- здесь ещё пусто;
- здесь уже стоит `X`;
- здесь уже стоит `O`.

---

## Теория 1: Состояние клетки и текущий игрок

Чтобы крестики-нолики были похожи на настоящую игру, недостаточно просто вывести символ.
Игра должна помнить, кто ходит сейчас и что уже лежит в каждой клетке.

Массив `board` по-прежнему хранит состояние поля.
Теперь читаем его так:

- `0` — пусто
- `1` — X
- `2` — O

Также игре нужна переменная, которая хранит, чей сейчас ход:

```csharp
public int currentPlayer = 1;
```

И ещё одна полезная переменная:

```csharp
public bool isGameOver;
```

Тип `bool` нужен, когда переменная хранит только два варианта ответа:

- да или нет;
- true или false;
- включено или выключено.

Она хранит очень простой ответ:

- `false` — партия ещё идёт;
- `true` — партия уже закончилась.

---

Текст в кнопке нужен игроку, а массив нужен логике.
Если в клетке написано `X`, человеку это видно сразу.
Но для проверки правил и принятия решений игре удобнее работать с числами в массиве.

### Шаг 2: Обновите `UpdateCell(int cellIndex)`

Раньше метод всегда ставил `xSymbol`. Теперь он должен смотреть на значение в массиве и ставить нужный символ.

Найдите метод `UpdateCell` и замените его содержимое:

```csharp
public void UpdateCell(int cellIndex)
{
    BoardCell boardCell = boardCells[cellIndex];

    if (board[cellIndex] == 1)
    {
        boardCell.SetSymbol(xSymbol);
    }
    else if (board[cellIndex] == 2)
    {
        boardCell.SetSymbol(oSymbol);
    }
}
```

> **Как работает по шагам:**
>
> 1. `boardCells[cellIndex]` — берём нужную клетку из массива по индексу.
> 2. `if (board[cellIndex] == 1)` — если в массиве `1`, значит ходил X.
> 3. `else if (board[cellIndex] == 2)` — если `2`, значит ходил O.
> 4. `SetSymbol(...)` — передаём в клетку нужную строку.

### Шаг 3: Добавьте `SwitchPlayer()`

Этот метод должен менять `currentPlayer` после каждого хода.

Добавьте метод в класс `TicTacToeGame`:

### Шаг 4: Сделайте `MakeMove(int cellIndex)`

Этот метод должен:

1. записать `currentPlayer` в массив;
2. обновить визуал клетки;

Добавьте метод в класс `TicTacToeGame`:

public void MakeMove(int cellIndex)
{

## Как остановить неправильный ход?

Игрок не должен:

- нажимать на уже занятую клетку;
- ломать ход случайным повторным кликом.

Значит, до выполнения хода игра должна сделать несколько маленьких проверок.

---

## Теория 2: Проверки перед ходом и маленькие методы-действия

Когда правила уже появились, игре нужно научиться не только делать ход, но и вовремя его останавливать.
Именно здесь становятся полезны короткие методы с одной задачей и ранний выход.

Очень удобно, когда у каждого метода есть одна понятная задача:

- `HandleCellClicked(...)` — получает клик;
- `MakeMove(...)` — делает ход;
- `SwitchPlayer()` — переключает игрока;
- `SetInteractable(...)` — включает или выключает клетку.

Это методы, которые **выполняют действие**.

Слово `interactable` в Unity можно читать очень просто:

- `true` — кнопку можно нажимать;
- `false` — кнопку нажимать нельзя.

---

Перед ходом игра может сделать ранний выход:

```csharp
if (board[cellIndex] != 0)
{
    return;
}
```

Это означает:

- если клетка уже занята;
- дальше код выполнять не нужно.

## Пояснение (ранний выход)

- Ранний выход делает код чище.

## Практика 2: Ограничиваем неправильные ходы

Теперь сделаем игру аккуратнее: занятую клетку нельзя нажать повторно.
После этой практики партия будет ощущаться заметно честнее и понятнее для игрока.

---

### Шаг 1: Добавьте проверки в `HandleCellClicked(int cellIndex)`

Откройте `TicTacToeGame.cs`, найдите метод `HandleCellClicked`.
В начале метода, **перед** вызовом `MakeMove(cellIndex)`, добавьте три проверки:

```csharp
public void HandleCellClicked(int cellIndex)
{
    if (isGameOver)
    {
        return;
    }

    if (cellIndex < 0 || cellIndex >= board.Length)
    {
        return;
    }

    if (board[cellIndex] != 0)
    {
        return;
    }

    MakeMove(cellIndex);
}
```

> **Порядок проверок важен:**
>
> 1. Сначала проверяем, не закончилась ли партия — тогда ни один клик вообще не работает.
> 2. Потом проверяем, что индекс допустимый (защита от случайных ошибок).
> 3. Потом проверяем, пуста ли клетка — нельзя ходить дважды в одно место.
> 4. Только после всех проверок делаем реальный ход.
