# Модуль 4, Урок 4: Крестики-нолики — Random.Range, бот и временный режим игры

**Цель урока:** познакомиться с `Random.Range`, научиться собирать список свободных клеток и сделать простого бота, который умеет выбирать допустимый ход во временной версии игры против компьютера.

После прошлого урока у нас уже есть полноценная игра на двоих.
Теперь добавим новый игровой опыт: партия против компьютера, который тоже умеет делать ходы по правилам.

---

## Как сделать, чтобы в крестики-нолики можно было играть не только вдвоём?

Игра уже умеет:

- ставить `X` и `O`;
- проверять победу;
- определять ничью.

Теперь хочется, чтобы у игрока появился противник.

---

Компьютерному противнику тоже нужно принимать решения:

- куда ходить;
- как не попасть в занятую клетку;
- как выбрать один вариант из нескольких.

---

## Как игре выбрать одну клетку из нескольких свободных?

Если на поле осталось несколько свободных клеток, боту нужно выбрать одну из них.
Сделать это можно через случайность.

Но сначала игра должна понять:

- какие клетки вообще свободны;
- из чего можно выбирать.

---

## Теория 1: Как случайность работает в играх

Бот тоже должен принимать решения, но для первого варианта игры ему не нужна сложная стратегия.
Ему достаточно уметь выбирать один допустимый ход из нескольких свободных.

В играх случайность используют очень часто:

- для выбора награды;
- для случайного поведения врага;
- для появления объектов;
- для выбора одного варианта из нескольких.

В Unity для этого часто используют:

```csharp
Random.Range(0, freeCells.Count)
```

Здесь `freeCells.Count` показывает, сколько сейчас свободных клеток лежит в списке.
Это почти то же самое, что `Length` у массива: и `Count`, и `Length` помогают узнать, сколько элементов сейчас хранится внутри.
То есть игра выбирает случайное число не «вообще», а только в пределах реального количества вариантов.

---

Но у случайности есть важное правило:

сначала нужно собрать **только допустимые варианты**,
и только потом выбирать из них случайный.

Если бот будет брать число сразу из `0..8`, он может попробовать пойти в уже занятую клетку.

### Шаг 2: Добавьте `GetFreeCells(int[] board)`

Этот метод проходит по массиву поля и собирает индексы всех пустых клеток в список.

Добавьте метод в класс `TicTacToeBot`:

```csharp
public List<int> GetFreeCells(int[] board)
{
    List<int> freeCells = new List<int>();

    for (int i = 0; i < board.Length; i++)
    {
        if (board[i] == 0)
        {
            freeCells.Add(i);
        }
    }

    return freeCells;
}
```

> **Как работает по шагам:**
>
> 1. Создаём пустой список `freeCells`.
> 2. Проходим по каждому элементу массива `board`.
> 3. Если элемент равен `0` — клетка пустая, добавляем её индекс в список через `Add(i)`.
> 4. Возвращаем готовый список свободных клеток.

```check
  - contains: "List<int>"
  - contains: "freeCells.Add(i)"
```

### Шаг 3: Добавьте `GetRandomMove(List<int> freeCells)`

Этот метод получает список свободных клеток и возвращает **один случайный индекс**:

```csharp
public int GetRandomMove(List<int> freeCells)
{
    int randomIndex = Random.Range(0, freeCells.Count);
    return freeCells[randomIndex];
}
```

> **Как бот выбирает ход:**
>
> 1. В `freeCells` лежат индексы всех пустых клеток (например, `[0, 3, 5]`).
> 2. `Random.Range(0, freeCells.Count)` выбирает случайное число в диапазоне `[0, Count)` — то есть `0`, `1` или `2` для трёх элементов.
> 3. `freeCells[randomIndex]` берёт реальный индекс клетки на поле по этому номеру.
> 4. Снаружи этот индекс можно передать в `MakeMove`, чтобы сделать ход за бота.
>
> **Важно:** `Random.Range(int, int)` для целых чисел возвращает число от первого аргумента **включительно** до второго **не включая** его. То есть `Random.Range(0, 3)` даст `0`, `1` или `2`, но никогда `3`.

### Шаг 4: Добавьте `ChooseMove(int[] board)`

Сделайте главный метод бота, который объединяет два предыдущих:

```csharp
public int ChooseMove(int[] board)
{
    List<int> freeCells = GetFreeCells(board);

    if (freeCells.Count == 0)
    {
        return -1;
    }

    return GetRandomMove(freeCells);
}
```

> **Зачем возвращать `-1`?** Если на поле нет свободных клеток, боту некуда ходить. Значение `-1` — это специальный «сигнал», что ход невозможен. Снаружи игра проверит это через `if (botMove >= 0)` и не будет вызывать `MakeMove` с некорректным индексом.

```check
rules:
  - filename: "TicTacToeBot.cs"
  - contains: "ChooseMove"
  - contains: "if (freeCells.Count == 0)"
  - contains: "return -1"
```

## Теория 2: Методы могут возвращать не только bool

В прошлых уроках методы уже возвращали ответ `да/нет`.
Теперь на игровом примере хорошо видно, что метод может вернуть и список вариантов, и один конкретный ход.

На прошлом уроке были методы, которые возвращали `bool`:

- `CheckWinner(...)`
- `MatchesLine(...)`
- `IsBoardFull()`

В этом уроке появятся и другие варианты:

- метод может вернуть `List<int>`;
- метод может вернуть `int`.

---

Это удобно читать так:

- `GetFreeCells(...)` возвращает список свободных клеток;
- `GetRandomMove(...)` возвращает один выбранный индекс;
- `ChooseMove(...)` возвращает итоговый ход бота.

То есть методы могут возвращать не только ответ `да/нет`, но и полезные данные.

## Когда бот должен делать ход?

Бот не должен ходить в любой момент.
Игра должна вызывать его только тогда, когда:

- ход игрока уже закончился;
- партия ещё не завершена;
- сейчас очередь бота.

---

## Теория 3: Как подключить бота к уже готовой логике

Когда бот уже умеет выбирать ход, остаётся самый важный шаг: встроить его в игру так, чтобы он не ломал готовые правила партии.
То есть бот должен использовать уже существующую логику, а не жить отдельно от неё.

Удобный порядок такой:

1. игрок делает ход;
2. `MakeMove(...)` обновляет поле;
3. игра проверяет победу и ничью;
4. если партия продолжается, а сейчас очередь бота, вызывается `MakeBotMove()`.

---

Внутри `MakeBotMove()` логика простая:

1. бот просит у себя ход через `ChooseMove(board)`;
2. если индекс корректный, игра делает ход через обычный `MakeMove(...)`.

Так бот не пишет правила игры заново, а использует уже существующую логику партии.

### Шаг 2: Добавьте `MakeBotMove()` в `TicTacToeGame`

Добавьте метод:

```csharp
public void MakeBotMove()
{
    int botMove = bot.ChooseMove(board);

    if (botMove >= 0)
    {
        MakeMove(botMove);
    }
}
```

> **Как работает:**
>
> 1. `bot.ChooseMove(board)` — спрашиваем у бота, в какую клетку он хочет пойти. Получаем индекс.
> 2. `if (botMove >= 0)` — проверяем, что индекс допустимый (не `-1`).
> 3. `MakeMove(botMove)` — делаем ход за бота через уже готовую логику игры.
>
> Бот не знает ничего о победе, ничьей или очерёдности. Всё это уже есть в `MakeMove`.

```check
  - filename: "TicTacToeGame.cs"

  - contains: "bot.ChooseMove(board)"
  - contains: "MakeMove(botMove)"
```
