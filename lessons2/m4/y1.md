**Цель урока:** собрать поле `3 x 3`, понять, что такое массив и индекс, связать UI-клетку с кодом и сделать первый рабочий клик по клетке.

В этом уроке мы начинаем игру с самого понятного и игрового результата: собираем поле и делаем так, чтобы нажатие по клетке действительно что-то меняло на экране.

## Как сделать, чтобы каждая клетка понимала: нажали именно на неё?

В `Крестиках-ноликах` игрок нажимает на одну из 9 клеток.
И игра должна понять две вещи:

- **куда именно нажали**;
- **что теперь показать в этой клетке**.

---

Если игра не понимает, по какой клетке кликнули, она не сможет правильно работать:

- не поймёт, где поставить символ;
- не сможет хранить состояние поля;
- не сможет потом проверять победу и ничью.

---

Сегодня закладываем фундамент, на котором дальше строится вся игра:

- поле;
- клетки;
- массив;
- первый рабочий клик.

## Как игре запомнить сразу всё поле, а не одну клетку?

Если в игре 9 клеток, нам нужно хранить состояние не одной кнопки, а всего поля целиком.
И желательно так, чтобы код не превратился в длинный и запутанный список переменных.

## Теория 1: Почему поле удобно хранить в массиве

Чтобы игра вообще могла запоминать ходы, ей нужно место, где хранится состояние всех клеток сразу.
Именно поэтому сейчас нам нужен массив, а не набор случайных отдельных переменных.

Сначала разберёмся, что такое **массив**.

Массив — это переменная, которая хранит **сразу много однотипных значений в одном месте**.

Представьте себе ряд из коробок с номерами:

- в каждой коробке можно что-то хранить;
- у каждой коробки есть свой номер;
- по номеру можно быстро найти нужную коробку.

В программировании массив работает похоже:

- он хранит несколько значений одного типа;
- у каждого элемента есть свой номер;
- этот номер называется **индексом**.

Например, если у нас есть 9 клеток поля, мы можем не создавать 9 отдельных переменных, а хранить всё в одном массиве.

---

Если бы мы пытались хранить клетки отдельно, получилось бы что-то неудобное вроде:

```csharp
int cell0;
int cell1;
int cell2;
// и так далее
```

Такой код сложно расширять и проверять.
Гораздо удобнее хранить поле `3 x 3` как один массив из 9 элементов:

```csharp
int[] board = new int[9];
```

Здесь:

- `int` означает, что внутри хранятся числа;
- `[]` показывает, что это массив;
- `new int[9]` создаёт массив на 9 элементов.

---

Очень важный момент:

- в C# массивы начинаются с индекса `0`, а не с `1`.

Поэтому первый элемент массива — это не `board[1]`, а именно:

```csharp
board[0]
```

Это непривычно только в начале.
Но в программировании так устроено очень многое, поэтому к этому важно привыкнуть сразу.

Слово `new` здесь нужно потому, что мы **создаём новый объект в памяти**.

То есть:

- без `new` мы просто сказали бы, что хотим переменную типа "массив чисел";
- с `new` мы реально создаём этот массив и говорим, сколько в нём будет элементов.

Можно читать строку так:

- `int[] board` — "я хочу переменную `board`, которая будет массивом чисел";
- `= new int[9]` — "создай новый массив из 9 чисел и положи его в `board`".

---

На первом уроке нам достаточно такого правила:

- `0` — клетка пустая.

То есть:

- `board[0]` — первая клетка;
- `board[1]` — вторая клетка;
- `board[2]` — третья клетка;
- и так далее.

Главная идея урока:

- каждая клетка имеет свой индекс;
- игра получает индекс клетки;
- по индексу можно менять состояние поля и визуал.

## Теория 2: Как клетка сообщает игре, куда нажали

Теперь у нас уже есть поле, но без связи между кнопкой и скриптом игра всё ещё "слепая".
Нужно объяснить ей, как одно нажатие превращается в понятный номер клетки.

Одна клетка должна знать:

- свой индекс `cellIndex`;
- где её `Button`;
- где текст символа `TMP_Text`;
- какой главный скрипт игры должен получить нажатие.

Для этого нужен отдельный скрипт клетки, например `BoardCell.cs`.

Идея простая:

```csharp
game.HandleCellClicked(cellIndex);
```

То есть клетка не решает правила игры сама.
Она только сообщает: "нажали на индекс 4".

Такое разделение очень полезно:

- `BoardCell` отвечает за нажатие и визуал;
- `TicTacToeGame` отвечает за состояние поля.

### Шаг 2: Добавьте `Awake()` и `OnDestroy()`

Теперь пусть клетка сама найдёт свои компоненты и подпишется на клик:

```csharp
public void Awake()
{
    button = GetComponent<Button>();
    symbolText = GetComponentInChildren<TMP_Text>();
    button.onClick.AddListener(OnCellClicked);
}

public void OnDestroy()
{
    button.onClick.RemoveListener(OnCellClicked);
}
```

Идея тут такая:

- в `Awake()` клетка готовится к работе;
- через `AddListener(...)` мы говорим кнопке, какой метод вызвать при нажатии;
- в `OnDestroy()` убираем подписку аккуратно.

### Шаг 6: Добавьте `HandleCellClicked(int cellIndex)`

Теперь игра должна:

1. получить индекс клетки;
2. проверить, что индекс корректный;
3. записать `1` в массив;
4. обновить клетку.

```csharp
public void HandleCellClicked(int cellIndex)
{
    if (cellIndex < 0 || cellIndex >= board.Length)
    {
        return;
    }

    if (board[cellIndex] != 0)
    {
        return;
    }

    board[cellIndex] = 1;
    UpdateCell(cellIndex);
}
```
